<task type="auto">
  <name>Task 2: Create Logo component with three SVG size variants</name>
  <files>frontend/components/logo.tsx</files>
  <action>
Create `frontend/components/logo.tsx` exporting a single `Logo` component with a `size` prop that controls which SVG variant renders.

**TypeScript interface:**
```tsx
interface LogoProps {
  size: 'small' | 'medium' | 'large'
  className?: string
}
```

Export as named export: `export function Logo({ size, className }: LogoProps)`

**Wrapper element:**
- Use `<svg>` with `role="img"` and `aria-label="ShipSecure"`
- Each variant gets its own `viewBox` dimensions
- Do NOT set fixed `width` or `height` on the SVG — let the parent control sizing via CSS/Tailwind classes
- Pass `className` through to the SVG element for external sizing control

**Small variant (size='small') — Shield-shaped S lettermark:**
- viewBox: `0 0 32 32`
- Draw an angular, geometric letter "S" that has shield-like geometry (sharp angles, faceted edges — "digital fortress" feel)
- The S should be recognizable as related to the full shield mark — same angular DNA
- Fill with `var(--color-brand-primary)` (the S IS the mark at small size, not a cutout)
- This variant is intended for 16px favicon contexts, so keep paths simple and bold

**Medium variant (size='medium') — Geometric shield mark with checkmark cutout:**
- viewBox: `0 0 32 36` (shields are taller than wide)
- Draw a geometric/angular shield shape with 2-3 facets. Sharp angles, not rounded — "digital fortress" feel
- The shield should have clear faceted geometry (e.g., a V-shaped bottom point, angular shoulders, subtle facet lines)
- Use `fill-rule="evenodd"` with a compound path: outer path is the shield silhouette, inner subpath is the checkmark
- The checkmark cuts out from the shield body as negative space (white shows through from background)
- Shield fill: `var(--color-shield-fill)` (brand blue, adapts in dark mode)
- Do NOT use a separate white path for the checkmark — use evenodd fill-rule so the checkmark is a true cutout
- Keep the checkmark simple and bold enough to read at 32px

**Large variant (size='large') — Shield mark + wordmark:**
- viewBox: `0 0 200 36` (wide to accommodate shield + text)
- Left side: Same shield mark as medium variant (reuse the same path data, positioned at left)
- Right side: "ShipSecure" wordmark as SVG path data (NOT an SVG `<text>` element — convert to paths for font portability)
- Wordmark typography: Inter medium/semi-bold weight aesthetic. Convert text to paths.
- Wordmark fill: `currentColor` (inherits from parent, automatically adapts to light/dark mode)
- Shield fill: `var(--color-shield-fill)` (same as medium)
- Gap between shield and wordmark: approximately 8-12 units (use your judgment for visual balance)

**Important constraints:**
- NO `<text>` elements anywhere (breaks if Inter font not loaded, per research anti-pattern)
- NO fixed width/height on SVG (prevents responsive scaling, per research anti-pattern)
- NO hard-coded hex colors (prevents theme adaptation, per research anti-pattern)
- NO separate mask/clipPath for the checkmark cutout (unnecessary complexity, per research)
- NO gradient on the shield (user decision: "flat solid color, no gradient")
- The component must be a single file with all three variants — NOT separate component files (per research)
- Use `'use client'` directive only if needed (this is pure SVG rendering, likely not needed)

**SVG path design guidance:**
For the shield shape, aim for something like:
- Top: flat or slightly curved top edge with angular shoulder points
- Sides: straight or slightly tapered sides with 1-2 facet angles
- Bottom: comes to a sharp V-point (classic shield bottom)
- Overall feel: technical, geometric, digital fortress — NOT rounded/friendly/cartoon

For the checkmark cutout:
- Simple two-stroke checkmark (short left stroke, long right stroke)
- Positioned center-right within the shield body
- Bold enough line width that it reads at 32px
