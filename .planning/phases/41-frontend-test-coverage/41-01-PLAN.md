  <action>
Create `frontend/__tests__/components/ScanHistoryTable.test.tsx`.

This is the most complex component — has a desktop table, mobile cards, severity badges, tier badges, expiry formatting, pagination, and clickable/non-clickable row logic. Focus on observable behavior, not implementation details.

**Create mock data factory** at the top of the file:

```typescript
import type { ScanHistoryItem } from '@/lib/types'

function makeScan(overrides: Partial<ScanHistoryItem> = {}): ScanHistoryItem {
  return {
    id: 'scan-1',
    target_url: 'https://example.com',
    status: 'completed',
    results_token: 'tok-abc',
    expires_at: new Date(Date.now() + 30 * 86400000).toISOString(),
    tier: 'authenticated',
    created_at: '2026-02-15T12:00:00Z',
    critical_count: 1,
    high_count: 2,
    medium_count: 0,
    low_count: 0,
    ...overrides,
  }
}
```

**Test groups:**

1. **Empty state**: Pass `scans=[]`. Assert "No completed scans yet." text is rendered.

2. **Basic row rendering**: Pass one scan. Assert:
   - Domain hostname "example.com" is rendered
   - Date "Feb 15, 2026" is rendered
   - Tier badge "Enhanced" is rendered (for `tier='authenticated'`)

3. **Severity badges**: Pass scan with `critical_count=2, high_count=1, medium_count=3, low_count=0`. Assert "2" and "1" and "3" appear. Assert zero-count severity badges are NOT rendered (SeverityBadge returns null when count is 0).

4. **Tier badges**:
   - `tier='authenticated'` renders "Enhanced"
   - `tier='anonymous'` renders "Basic"

5. **Expiry states**:
   - Future expiry (> 3 days): renders "X days left" text
   - Expiring soon (<= 3 days, > 0): renders "X days left" text (with caution styling, but we test text)
   - Expired (<= 0 days): renders "Expired" text
   - Failed scan: renders "Failed" text
   - Null expires_at: renders em-dash character

6. **Clickable rows**: A completed scan with `results_token` and non-expired `expires_at` should have a link to `/results/tok-abc`. Use `screen.getByRole('link', { name: /view results/i })` to find the overlay link.

7. **Non-clickable rows**: An expired scan or a scan with `results_token=null` should NOT have a results link. Use `screen.queryByRole('link', { name: /view results/i })` and assert it is null.

8. **Pagination**:
   - `totalPages=1` should NOT render pagination links
   - `totalPages=3, currentPage=2` should render page number links. Assert page "1" and "3" are links (via `screen.getByRole('link', { name: '1' })` or by text), and "2" is bold (current page, not a link).

9. **Multiple scans**: Pass 2-3 scans with different IDs. Assert all domains are rendered.

**Important notes for the executor:**
- The component uses `Date.now()` internally for expiry calculations. Use `vi.useFakeTimers()` and `vi.setSystemTime()` to make date-dependent tests deterministic. Remember to call `vi.useRealTimers()` in `afterEach`.
- The component uses `next/link` which is already mocked globally (renders as `<a>` tag).
- SeverityBadge returns null for count=0 — this is by design.
- The extractHostname helper parses URLs — test implicitly by asserting "example.com" appears (not "https://example.com").
