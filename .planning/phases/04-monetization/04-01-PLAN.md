  <files>migrations/20260206000001_add_paid_audits.sql, Cargo.toml</files>
  <action>
Create migration file `migrations/20260206000001_add_paid_audits.sql` with:

1. `paid_audits` table:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `scan_id UUID NOT NULL REFERENCES scans(id) ON DELETE CASCADE`
   - `stripe_checkout_session_id VARCHAR(255) NOT NULL UNIQUE`
   - `stripe_payment_intent_id VARCHAR(255)` (nullable, populated by webhook)
   - `amount_cents INTEGER NOT NULL`
   - `currency VARCHAR(3) DEFAULT 'usd'`
   - `customer_email VARCHAR(255) NOT NULL`
   - `status VARCHAR(20) NOT NULL DEFAULT 'pending'` (pending, completed, failed, refunded)
   - `pdf_generated_at TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ DEFAULT NOW()`
   - `updated_at TIMESTAMPTZ DEFAULT NOW()`
   - Indexes on: scan_id, stripe_checkout_session_id, status

2. `stripe_events` table (webhook idempotency):
   - `event_id VARCHAR(255) PRIMARY KEY` (Stripe event ID like evt_xxx)
   - `processed_at TIMESTAMPTZ DEFAULT NOW()`

3. Add `tier` column to scans:
   - `ALTER TABLE scans ADD COLUMN tier VARCHAR(10) DEFAULT 'free' CHECK (tier IN ('free', 'paid'));`

Add dependencies to Cargo.toml:
```toml
async-stripe = { version = "0.41", default-features = false, features = ["runtime-tokio-hyper", "checkout"] }
genpdf = "0.2"
hmac = "0.12"
sha2 = "0.10"
hex = "0.4"
```

Note: hmac/sha2/hex are for Stripe webhook signature verification (manual verification is more reliable than async-stripe's built-in Webhook::construct_event which has version-dependent issues). genpdf is for Plan 04 (PDF generation) but adding now avoids Cargo.toml conflicts.
  </action>

Run `cargo check` to verify Cargo.toml compiles with new dependencies. Verify migration SQL syntax is valid by inspecting the file.
  </verify>
