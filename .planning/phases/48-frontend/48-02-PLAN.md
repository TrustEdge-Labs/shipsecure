## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Backend (results fetch) | Token in URL is user-controlled; backend validates |
| sessionStorage (inline results) | Client-only data for share-unavailable fallback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-05 | Spoofing | results/[token]/page.tsx | mitigate | Token is 256-bit random (43 chars base64). Infeasible to guess. Backend validates token exists. |
| T-48-06 | Tampering | sessionStorage inline results | accept | Client-only fallback for share-unavailable case. No security impact — user sees their own results. No server trust of this data. |
| T-48-07 | Information Disclosure | supply-chain-findings.tsx | accept | OSV IDs and descriptions are public data from osv.dev. No PII. No gating needed (unlike web app findings). |
| T-48-08 | Repudiation | Plausible events | accept | Analytics events are fire-and-forget. No audit trail needed. |
</threat_model>
