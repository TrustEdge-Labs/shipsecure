<objective>
Implement PDF report generation and email delivery with PDF attachment for paid audits.

Purpose: Paid audit customers receive a professional branded PDF report attached to their completion email. The PDF includes an executive summary (grade, URL, date, finding counts), all findings organized by severity with descriptions and remediation guidance, and a section on detected framework/platform. The email is sent via the existing Resend integration with an added base64-encoded PDF attachment.

Output: `src/pdf.rs` module for in-memory PDF generation, updated `src/email/mod.rs` with PDF attachment support.

1. Download Liberation Sans font files for genpdf. Create `fonts/` directory at project root. genpdf requires TTF font files. Download LiberationSans-Regular.ttf, LiberationSans-Bold.ttf, LiberationSans-Italic.ttf from the Liberation Fonts project (https://github.com/liberationfonts/liberation-fonts/releases). Place them in `fonts/`.

   If downloading is not possible, use genpdf's built-in font handling. Check genpdf docs — the `fonts::from_files` function loads from a directory. Alternatively, if Liberation fonts aren't available, use a system font path or bundle the fonts.

   Simplest approach: Try `fonts::from_files("./fonts", "LiberationSans", None)`. If fonts aren't available at runtime, fall back to creating a basic PDF using printpdf's built-in fonts (genpdf wraps printpdf which has built-in Helvetica/Courier).

   Actually, the most reliable approach: Use genpdf's `fonts::from_files` but ALSO handle the case where fonts are missing gracefully. For development, create a `fonts/` directory with a `.gitkeep` and document that Liberation Sans TTFs need to be placed there. In the generate_report function, return an error if fonts are missing.

2. Create `src/pdf.rs` with a `generate_report` function:

```rust
pub fn generate_report(
    target_url: &str,
    grade: &str,
    scan_date: &str,

    platform: Option<&str>,
```

PDF layout (using genpdf):
- **Page 1 — Title and Executive Summary:**
  - Title: "TrustEdge Security Audit Report" (bold, 24pt)
  - Subtitle: "Deep Audit Report" (16pt)
  - Line separator
  - Target URL, Scan Date, Overall Grade
  - Framework/Platform if detected
  - Executive Summary paragraph: "This deep security audit identified {N} vulnerabilities across {severity_count} severity levels. {critical_count} critical and {high_count} high-severity findings require immediate attention."
  - Findings count by severity (as bullet list):
    - Critical: N
    - High: N
    - Medium: N
    - Low: N

- **Subsequent pages — Findings by Severity:**
  - Section header per severity level (bold, 16pt): "CRITICAL SEVERITY FINDINGS" etc.
  - Skip severity levels with 0 findings
  - For each finding:
    - Title (bold, 12pt)
    - Scanner: {scanner_name}
    - Description paragraph
    - "Remediation:" (italic) followed by remediation text
    - Separator line or break between findings
  - Mark vibe-code findings with "[VIBE-CODE]" prefix in title

- **Last page — Remediation Roadmap:**
  - "Remediation Priority Roadmap" heading
  - "1. Address Critical findings immediately"
  - "2. Schedule High findings for this week"
  - "3. Plan Medium findings for next sprint"
  - "4. Track Low findings in backlog"
  - Footer: "Generated by TrustEdge Audit — trustedgeaudit.com"

Use genpdf's auto-pagination. Set margins to 15mm. Line spacing 1.25.

Error type: Create `PdfError` enum with FontError, RenderError variants.

3. Update `src/lib.rs`:
   - Add `pub mod pdf;`

IMPORTANT: genpdf 0.2 API uses `elements::Paragraph`, `elements::Break`, `style::Style`, `fonts::from_files`. Check the actual API if compilation fails and adapt. The key patterns:
- `Document::new(font_family)` creates doc
- `doc.push(element)` adds content
- `doc.render(&mut writer)` writes to any Write impl (use Vec<u8>)
