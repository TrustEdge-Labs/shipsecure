**Request parsing (per D-01):**
Define an enum for input dispatch:
```rust
enum SupplyChainInput {
    GitHubUrl(String),          // JSON body with "github_url" field
    LockfileContent(String),    // JSON body with "lockfile_content" field
    FileUpload(String),         // multipart/form-data with "lockfile" field
}
```

The handler signature:
```rust
pub async fn create_supply_chain_scan(
    State(state): State<AppState>,
    ConnectInfo(addr): ConnectInfo<SocketAddr>,
    Extension(request_id): Extension<RequestId>,
    headers: axum::http::HeaderMap,
    body: axum::body::Body,
) -> Result<Json<serde_json::Value>, ApiError>
```

Accept raw `Body` and inspect `Content-Type` header to dispatch:
- If `content-type` starts with `multipart/form-data`: parse multipart using `axum::extract::Multipart` (construct from body + headers). Extract field named "lockfile", read its bytes (reject if >5MB per D-09), convert to String. Result: `SupplyChainInput::FileUpload(content)`.
- If `content-type` is `application/json`: read body bytes (reject if >5MB), deserialize to a helper struct with `github_url: Option<String>` and `lockfile_content: Option<String>`. If `github_url` is present, result: `SupplyChainInput::GitHubUrl(url)`. Else if `lockfile_content` present, result: `SupplyChainInput::LockfileContent(content)`. Else return 400 "Must provide github_url, lockfile_content, or upload a file".
- Otherwise: return 400 "Unsupported content type".

**GitHub URL fetching (per D-05, D-06, D-07):**
Create helper `async fn fetch_lockfile_from_github(url: &str) -> Result<String, SupplyChainError>`:
- Parse owner/repo from URL with regex: `^https?://github\.com/([^/]+)/([^/]+?)(?:\.git)?(?:/.*)?$`
- Construct raw URL: `https://raw.githubusercontent.com/{owner}/{repo}/main/package-lock.json`
- Use reqwest client with 5s timeout (per D-07). If `GITHUB_TOKEN` env var is set, add `Authorization: token {value}` header.
- If 404, retry with `master` instead of `main`.
- If both 404, return `SupplyChainError::GitHubFetch("No package-lock.json found on main or master branch")`.
- On success, check response body size <= 5MB, return body text.

**Abuse controls (per D-08, D-09):**
- Apply existing rate limiter: call `rate_limit::check_rate_limits()` with target_domain extracted from GitHub URL (or "upload"/"paste" for non-URL inputs). For non-URL inputs, use IP as the domain to avoid cache collisions.
- After parsing lockfile (before scan), check dep count via `lockfile_parser::parse()` result length. If >5000, return `SupplyChainError::DepCountExceeded(count)`.

The double-parse is negligible (~ms). Alternatively, expose a count-only function. Keep it simple: just wrap scan_lockfile in a check. **Handler flow:**
1. Extract optional Clerk user (per D-02): `extract_optional_clerk_user(&state, &headers).await`
2. Parse input from body (dispatch by content type)
3. If GitHubUrl: fetch lockfile content from GitHub
4. Validate lockfile content not empty
5. Pre-check dep count: `lockfile_parser::parse(&content)?` — if len > 5000, return DepCountExceeded
6. Create DB row: `db::scans::create_supply_chain_scan()` with target_url = github URL or "upload:{filename}" or "paste"
7. Wrap scan in 30s timeout (per D-03): `tokio::time::timeout(Duration::from_secs(30), scan_lockfile(&content))`
8. On timeout: mark scan failed, return 504
9. On scan error: mark scan failed, map error to HTTP (see below)
10. On success: generate token, attempt DB persist (per D-13, D-15)
11. Return results JSON

**Error-to-HTTP mapping (per D-04):**
Implement `From<SupplyChainError> for ApiError`:
```rust
impl From<SupplyChainError> for ApiError {
    fn from(err: SupplyChainError) -> Self {
        match err {
            SupplyChainError::LockfileParse(msg) => ApiError::Custom {
                status: StatusCode::BAD_REQUEST,
                error_type: "https://shipsecure.ai/errors/invalid-lockfile".to_string(),
                title: "Invalid Lockfile".to_string(),
                detail: format!("Invalid lockfile format: {msg}"),
            },
            SupplyChainError::OsvQuery(msg) => ApiError::Custom {
                status: StatusCode::BAD_GATEWAY,
                error_type: "https://shipsecure.ai/errors/osv-unavailable".to_string(),
                title: "Vulnerability Database Unavailable".to_string(),
                detail: "Vulnerability database unavailable, please try again".to_string(),
            },
            SupplyChainError::GitHubFetch(msg) => ApiError::Custom {
                status: StatusCode::BAD_GATEWAY,
                error_type: "https://shipsecure.ai/errors/github-fetch".to_string(),
                title: "GitHub Fetch Failed".to_string(),
                detail: msg,
            },
            SupplyChainError::ChunkFailure(msg) => ApiError::Custom {
                status: StatusCode::BAD_GATEWAY,
                error_type: "https://shipsecure.ai/errors/vuln-check-failed".to_string(),
                title: "Vulnerability Check Failed".to_string(),
                detail: "Vulnerability check failed, please try again".to_string(),
            },
            SupplyChainError::DepCountExceeded(n) => ApiError::Custom {
                status: StatusCode::BAD_REQUEST,
                error_type: "https://shipsecure.ai/errors/too-many-deps".to_string(),
                title: "Too Many Dependencies".to_string(),
                detail: format!("Too many dependencies ({n}, max 5000)"),
            },
            SupplyChainError::Timeout => ApiError::Custom {
                status: StatusCode::GATEWAY_TIMEOUT,
                error_type: "https://shipsecure.ai/errors/scan-timeout".to_string(),
                title: "Scan Timed Out".to_string(),
                detail: "Scan timed out, try a smaller lockfile".to_string(),
            },
        }
    }
}
```

**Result persistence with graceful fallback (per D-13, D-15, D-16):**
After successful scan:
1. Generate token using same pattern as worker_pool: `rand::thread_rng().gen::<[u8; 32]>()` -> base64 URL_SAFE_NO_PAD
2. Serialize `SupplyChainScanResult` to `serde_json::Value`
3. Attempt `db::scans::complete_supply_chain_scan()` with expires_at = Utc::now() + 30 days
4. If DB write succeeds: include `share_url: "/supply-chain/results/{token}"` and `results_token: token` in response
5. If DB write fails: log error, return results inline with `share_url: null` and `share_unavailable: true` — NEVER fail the scan response (per D-13)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Handler | Untrusted lockfile content, GitHub URLs, multipart uploads |
| Handler -> GitHub | Server-side fetch to raw.githubusercontent.com only |
| Handler -> OSV.dev | Via scan_lockfile(), already hardened in Phase 46 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47-03 | S (Spoofing) | GitHub URL | mitigate | Strict regex only allows github.com URLs; only fetch from raw.githubusercontent.com (per D-05) |
| T-47-04 | T (Tampering) | Lockfile content | accept | Lockfile is untrusted input parsed by serde_json; no code execution. Parser rejects invalid JSON. |
| T-47-05 | D (DoS) | Large uploads | mitigate | 5MB body limit (D-09), 5000 dep cap (D-09), 30s timeout (D-03), existing rate limiter (D-08) |
| T-47-06 | D (DoS) | GitHub SSRF | mitigate | Only raw.githubusercontent.com is fetched; no arbitrary URL fetch. GITHUB_TOKEN optional. |
| T-47-07 | I (Info Disclosure) | Error messages | mitigate | OsvQuery and ChunkFailure return generic messages; internal details logged only (per D-04) |
| T-47-08 | R (Repudiation) | Scan creation | accept | Scans logged to DB with IP and optional user_id; structured logging captures request_id |
</threat_model>
