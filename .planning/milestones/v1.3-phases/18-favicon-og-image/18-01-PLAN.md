  <action>
1. Install sharp and to-ico as devDependencies:
   ```bash
   cd frontend && npm install --save-dev sharp to-ico
   ```

2. Create `frontend/scripts/generate-favicons.mjs` (ESM script, NOT TypeScript — runs directly with node):
   ```javascript
   import sharp from 'sharp'
   import toIco from 'to-ico'
   import { writeFile } from 'node:fs/promises'
   import { join, dirname } from 'node:path'
   import { fileURLToPath } from 'node:url'

   const __dirname = dirname(fileURLToPath(import.meta.url))
   const root = join(__dirname, '..')
   const input = join(root, 'public', 'logo.png')

   async function generateFavicons() {
     // Generate 16x16 and 32x32 PNGs for ICO
     const png16 = await sharp(input)
       .resize(16, 16, { fit: 'contain', background: { r: 0, g: 0, b: 0, alpha: 0 } })
       .png()
       .toBuffer()

     const png32 = await sharp(input)
       .resize(32, 32, { fit: 'contain', background: { r: 0, g: 0, b: 0, alpha: 0 } })
       .png()
       .toBuffer()

     // Create multi-resolution ICO (32x32 first for primary display)
     const ico = await toIco([png32, png16])
     await writeFile(join(root, 'app', 'favicon.ico'), ico)
     console.log('Generated: app/favicon.ico (16x16 + 32x32)')

     // Generate Apple touch icon (180x180)
     await sharp(input)
       .resize(180, 180, { fit: 'contain', background: { r: 0, g: 0, b: 0, alpha: 0 } })
       .png()
       .toFile(join(root, 'app', 'apple-icon.png'))
     console.log('Generated: app/apple-icon.png (180x180)')
   }

   generateFavicons().catch(err => {
     console.error('Favicon generation failed:', err)
     process.exit(1)
   })
   ```

3. Run the script to generate favicon.ico and apple-icon.png:
   ```bash
   cd frontend && node scripts/generate-favicons.mjs
   ```

4. Verify the generated files exist and have reasonable sizes:
   ```bash
   file app/favicon.ico    # Should show "MS Windows icon resource"
   file app/apple-icon.png # Should show "PNG image data, 180 x 180"
   ```

IMPORTANT: The source logo is 1536x1024 (3:2 aspect ratio), so `fit: 'contain'` with transparent background will letterbox it properly within the square favicon dimensions. This is correct — the shield+wordmark will be centered with transparent padding.

Create `frontend/app/icon.svg` — a simplified shield shape that works at 16x16 to 32x32 sizes with dark mode color adaptation.

The SVG favicon CANNOT use the full logo.png (SVG favicons must be vector). Instead, create a simplified geometric shield inspired by the ShipSecure brand:

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32" width="32" height="32">
  <style>
    .shield { fill: #2563eb; }
    @media (prefers-color-scheme: dark) {
      .shield { fill: #3b82f6; }
    }
  </style>
  <path class="shield" d="M16 2 L28 8 L28 18 Q28 28 16 30 Q4 28 4 18 L4 8 Z"/>
</svg>
```

Color mapping from design tokens (hardcoded in SVG since CSS custom properties don't work in standalone SVG files):
- Light mode: `#2563eb` — corresponds to blue-600 (brand-primary in light mode, approximate hex for oklch(0.546 0.245 262))
- Dark mode: `#3b82f6` — corresponds to blue-500 (brand-primary in dark mode, approximate hex for oklch(0.623 0.214 259))

The shield path is a simple pointed-bottom shield that reads clearly at 16x16. Do NOT add text, checkmarks, or fine details — they turn to illegible mud at favicon sizes.

Next.js auto-detects `icon.svg` in `/app` and generates: `<link rel="icon" href="/icon.svg" type="image/svg+xml" />`
