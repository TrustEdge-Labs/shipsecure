## Vision

### 10x Check
The 10x version of customer acquisition isn't just a blog post and a polished funnel. It's becoming the definitive "first responder" to every vibe-code security incident. Every time a CVE drops, ShipSecure is the first scanner to publish results, the first to offer a targeted check page, and the first name developers think of when they wonder "is my app safe?" The content pipeline automates: CVE feed -> landing page -> social post -> scan results. The product markets itself through every security incident in the ecosystem.

**Note:** The automated CVE pipeline is aspirational (12-month vision). This plan's accepted scope builds the manual version (Weekend 2 content) and the infrastructure it would run on (/blog, /check/{platform}).

## Accepted Scope (final, after all decisions)
- Plausible custom events for demo-scan, signup, first-real-scan, share-click
- /blog route with MDX support for hosting CVE content
- /check/{platform} landing pages (Lovable, Bolt, v0) with pre-filled scan CTA
- Reopen anonymous scans for any URL (revert Juice Shop lockdown)
- Own-URL-as-demo: landing page is one input field, paste any URL, see results
- Per-target rate limiting: 5/domain/hour, cached results for repeat scans
- Drop domain verification requirement for authenticated scans
- Increase anonymous scan rate limit to 3/IP/day
- Share results button (copy URL + text OG meta tags)
- Expired results page: "Scan again" CTA (conversion opportunity)
- Empty blog: "Coming soon" page

## Security Notes (for /plan-eng-review)
Reopening anonymous scans is a security policy reversal. The eng review should verify:
- `src/ssrf/validator.rs` blocks RFC 1918, link-local, cloud metadata (169.254.169.254), and redirect chains
- Per-target rate limiter (5/domain/hour) prevents DDoS amplification
- Per-IP rate limiter increased to 3/IP/day (from 1/IP/24h)
- E2E tests updated to reflect new anonymous scan behavior
- Whether additional logging/alerting is needed for anonymous scan abuse patterns
