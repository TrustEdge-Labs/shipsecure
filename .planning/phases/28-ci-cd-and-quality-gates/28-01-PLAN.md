**1. Create `.github/workflows/ci.yml`:**

Create the CI workflow file with the following structure (per locked decisions):

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  unit-tests:
    name: Unit Tests
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-node@v6
        with:
          node-version: '22'
      - name: Cache node_modules
        id: cache
        uses: actions/cache@v5
        with:
          path: frontend/node_modules
          key: ${{ runner.os }}-node22-${{ hashFiles('frontend/package-lock.json') }}
          # No restore-keys — exact match only to prevent stale cache issues
      - name: Install dependencies
        if: steps.cache.outputs.cache-hit != 'true'
        run: npm ci
      - name: Run unit tests with coverage
        run: npm run test:ci

  e2e-tests:
    name: E2E Tests
    runs-on: ubuntu-latest
    needs: [unit-tests]
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-node@v6
        with:
          node-version: '22'
      - name: Cache node_modules
        id: cache
        uses: actions/cache@v5
        with:
          path: frontend/node_modules
          key: ${{ runner.os }}-node22-${{ hashFiles('frontend/package-lock.json') }}
      - name: Install dependencies
        if: steps.cache.outputs.cache-hit != 'true'
        run: npm ci
      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium
      - name: Build production app
        run: npm run build
        env:
          NEXT_PUBLIC_BACKEND_URL: http://localhost:3001
          NEXT_PUBLIC_SITE_URL: https://shipsecure.io
      - name: Run E2E tests
        run: npm run test:e2e
        env:
          CI: true
      - name: Upload test artifacts on failure
        if: failure()
        uses: actions/upload-artifact@v6
        with:
          name: playwright-artifacts-${{ github.run_id }}
          path: |
            frontend/playwright-report/
            frontend/test-results/
          retention-days: 7
```

Key details:
- `e2e-tests` has `needs: [unit-tests]` — runs sequentially, not fully parallel (per user decision, CI-03)
- Cache path is `frontend/node_modules` (repo-root-relative, not working-directory-relative — `actions/cache` path is relative to `$GITHUB_WORKSPACE`)
- No `restore-keys` on cache — exact key only to prevent stale node_modules
- `npm ci` skipped when cache hits (per locked decision)
- Playwright browsers installed fresh each run (per locked decision, no browser caching)
- Clean Next.js build each time (per locked decision)
- NEXT_PUBLIC env vars set at build time for E2E job (critical — these are inlined at build time)
- Artifact upload uses `if: failure()` — only on test failures, not on success

**2. Add coverage thresholds to `frontend/vitest.config.ts`:**

Add a `thresholds` block inside the existing `coverage` section:

```typescript
thresholds: {
  lines: 80,
  functions: 80,
  branches: 75,
},
```

Place it after the `exclude` array inside the `coverage` object. This makes `vitest run` exit with non-zero code when coverage is below thresholds, which naturally fails the CI job (QUAL-01, QUAL-03).

QUAL-02 is already satisfied — the existing `reporter: ['text', 'html', 'lcov']` generates HTML and lcov formats.

**3. Add screenshot capture to `frontend/playwright.config.ts`:**

Add `screenshot: 'only-on-failure'` to the `use:` block, after the `trace` line:

```typescript
use: {
  baseURL: E2E_BASE_URL,
  trace: 'on-first-retry',
  screenshot: 'only-on-failure',
},
```

This ensures screenshots are captured for every failing test and included in the uploaded artifacts (CI-05). Traces are already configured with `trace: 'on-first-retry'`.
