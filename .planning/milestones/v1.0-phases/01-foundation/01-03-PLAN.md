**src/orchestrator/worker_pool.rs:**
Implement `ScanOrchestrator` struct:

```rust
pub struct ScanOrchestrator {
    pool: PgPool,
    semaphore: Arc<Semaphore>,
    max_scanner_timeout: Duration,
}
```

Constructor: `pub fn new(pool: PgPool, max_concurrent: usize) -> Self`
- Create semaphore with max_concurrent permits (default 5)
- Set max_scanner_timeout to 60 seconds

Core method: `pub async fn execute_scan(&self, scan_id: Uuid, target_url: &str) -> Result<(), OrchestratorError>`

This method:
1. Acquires a semaphore permit (blocks if at max concurrency).
2. Updates scan status to InProgress in the database.
3. Runs all registered scanners against the target URL. For Phase 1, only `security_headers` scanner exists. Structure the code to easily add more scanners later:

```rust
let scanner_results = vec![
    self.run_scanner_with_retry("security_headers", target_url).await,
    // Future: self.run_scanner_with_retry("tls", target_url).await,
    // Future: self.run_scanner_with_retry("nuclei", target_url).await,
];
```

4. For each scanner, call `run_scanner_with_retry` which:
   a. Wraps the scanner call in `tokio::time::timeout(self.max_scanner_timeout, ...)`
   b. On first failure (timeout or error), log warning, retry ONCE
   c. On second failure, record the failure but continue (partial results)
   d. On success, collect Vec<Finding>

5. Flatten all findings from all scanners into one Vec<Finding>.
6. Run aggregator::deduplicate_findings on the combined findings.
7. Compute score via aggregator::compute_score.
8. Persist findings to database via db::findings::insert_findings.
9. Update scan status to Completed with the computed score.
10. If ALL scanners failed, update scan status to Failed with error message.
11. Release semaphore permit (automatic via drop).

Helper: `async fn run_scanner_with_retry(&self, name: &str, url: &str) -> ScannerResult`

Define `ScannerResult` as:
```rust
pub struct ScannerResult {
    pub scanner_name: String,
    pub findings: Vec<Finding>,
    pub error: Option<String>,
}
```

Define `OrchestratorError` enum: DatabaseError(sqlx::Error), AllScannersFailed(String).

Also implement a convenience method for the API to use:
`pub fn spawn_scan(&self, scan_id: Uuid, target_url: String)`
- Clones self (wrap internals in Arc if needed), spawns a tokio task that calls execute_scan
- Fire-and-forget from the API's perspective (scan runs in background)
- Logs errors from execute_scan but doesn't propagate them (the scan status in DB reflects the outcome)

IMPORTANT: The semaphore MUST be acquired before spawning the task, not inside the task. This ensures backpressure -- if all permits are taken, the spawn_scan call blocks until one is available. Actually, for a fire-and-forget spawn, acquire INSIDE the spawned task so the API handler doesn't block. The semaphore still limits concurrency, just with a queue of spawned tasks waiting for permits.
