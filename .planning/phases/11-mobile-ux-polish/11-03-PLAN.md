    Audit and optimize for Lighthouse performance. The current codebase has no images (so no LCP image optimization needed), uses Google Fonts (Inter via next/font), and has Plausible analytics loaded afterInteractive. Key optimizations:

    **frontend/app/layout.tsx:**
    1. Verify the viewport meta tag is properly configured. Next.js App Router auto-generates viewport from exported `viewport` config, but we should export it explicitly for clarity. Add a viewport export:
       ```typescript
       export const viewport = {
         width: 'device-width',
         initialScale: 1,
         themeColor: '#ffffff',
       }
       ```
       This is the Next.js 14+ way to set viewport (not via meta tag in head).

    2. The Inter font is already loaded via `next/font/google` which is optimal (auto font-display: swap, self-hosted). No change needed.

    3. The Plausible scripts use `strategy="afterInteractive"` which is correct -- they don't block rendering.

    4. Add `suppressHydrationWarning` to the `<html>` tag to prevent hydration warnings from dark mode or browser extensions (common Lighthouse issue): `<html lang="en" suppressHydrationWarning>`.

    **frontend/app/globals.css:**
    - The current CSS is minimal (just `@import "tailwindcss"` and custom properties). This is fine.
    - Ensure the body styles don't cause CLS. The current `font-family: var(--font-inter), system-ui, -apple-system, sans-serif` with Inter loaded via next/font is optimal because Inter is loaded as a variable font with `font-display: swap`.

    **frontend/app/page.tsx:**
    - The landing page hero section uses `bg-gradient-to-r from-blue-600 to-blue-800 bg-clip-text text-transparent` for the heading. This is CSS-only, no performance concern.
    - The `getScanCount()` fetch uses `next: { revalidate: 60 }` which is good for ISR caching.
    - No images are used on the landing page, so LCP is the hero text. Text rendering is fast with self-hosted Inter font.
    - The JSON-LD scripts are inline and tiny, no impact.

    Run a Lighthouse audit after making changes:

    Additional performance check: ensure `overflow-x: hidden` is NOT applied to body (it masks horizontal scroll bugs instead of fixing them). If the styles look clean, no change needed.
