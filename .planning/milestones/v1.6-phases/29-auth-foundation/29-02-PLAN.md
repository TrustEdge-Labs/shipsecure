4. **frontend/proxy.ts** — Create the Next.js 16 middleware file (NOT middleware.ts):
   ```typescript
   import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

   const isProtectedRoute = createRouteMatcher(['/dashboard(.*)'])

   export default clerkMiddleware(async (auth, req) => {
     if (isProtectedRoute(req)) await auth.protect()
   })

   export const config = {
     matcher: [
       '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
       '/(api|trpc)(.*)',
     ],
   }
   ```
   **CRITICAL:** The file MUST be named `proxy.ts` (not `middleware.ts`). Next.js 16.1.6 uses `proxy.ts`. The function export stays as `export default`. The config export stays named `config`.

   - The ClerkProvider goes OUTSIDE the `<html>` tag per Clerk docs pattern:
     ```tsx
     return (
       <ClerkProvider>
         <html lang="en" suppressHydrationWarning>
           {/* ... existing content unchanged ... */}

       </ClerkProvider>

     ```
   - Use Clerk's default appearance — no custom theming per user decision.
