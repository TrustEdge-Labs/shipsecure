**FindingAccordion.test.tsx** — 8-10 tests:

Use a standard mock finding object:
```typescript
const mockFinding: Finding = {
  id: '1', title: 'Missing CSP header',
  description: 'Content-Security-Policy header is missing from responses.',
  severity: 'high', remediation: 'Add CSP header to your Next.js configuration.',
  scanner_name: 'security_headers', vibe_code: false,
}
```

Test groups:
1. **Initial Rendering (3 tests):**
   - Renders finding title in collapsed state — `screen.getByText(/missing csp header/i)` is in document
   - Shows severity badge — assert text matching severity (e.g., "high") is visible in uppercase badge
   - Description is NOT visible when collapsed — `screen.queryByText(/content-security-policy header is missing/i)` returns null (the element exists in DOM but has `max-h-0 opacity-0` via CSS — use `queryByText` and check that the content container has the collapsed class, OR check that "How to Fix" heading is not visible to the user). NOTE: Since CSS transitions don't work in happy-dom, the content IS technically in the DOM even when collapsed. Use `defaultExpanded={false}` (the default) and verify the expand/collapse toggle behavior rather than asserting visibility of the hidden content.

2. **Expand/Collapse Behavior (3 tests):**
   - Expands on click to show description — `const user = userEvent.setup()`, click the button with finding title, then verify the description text IS in the document. The component uses `max-h-[1000px] opacity-100` when expanded vs `max-h-0 opacity-0` when collapsed. Since happy-dom doesn't compute styles, test by rendering with `defaultExpanded={true}` and asserting content is present.
   - Renders with `defaultExpanded={true}` shows description immediately — pass prop, assert description text visible
   - Shows "How to Fix" heading and remediation text when expanded — render with `defaultExpanded={true}`, assert "How to Fix" and remediation text visible

3. **Metadata Display (2-3 tests):**
   - Shows vibe-code tag when `vibe_code: true` — create finding with `vibe_code: true`, assert "Vibe-Code" text visible
   - Does NOT show vibe-code tag when `vibe_code: false` — assert "Vibe-Code" text NOT in document via `queryByText`
   - Shows scanner display name — assert scanner name text is visible (e.g., "Headers" for scanner_name "security_headers"). Note: scanner display name uses a `hidden sm:inline` class, so it may or may not render depending on happy-dom viewport. Use `queryByText` with a fallback approach.

**Important note about expand/collapse testing:**
The FindingAccordion uses CSS `max-h-0 opacity-0` for collapse and `max-h-[1000px] opacity-100` for expand. In happy-dom, CSS transitions and computed styles don't work, so content is always technically in the DOM. The reliable test approach is:
- Test `defaultExpanded={true}` renders all content
- Test `defaultExpanded={false}` (default) then click toggles `isExpanded` state (which controls CSS classes)
- Test clicking the button toggles the state by checking the chevron rotation class (`rotate-180`)
