  <action>
Create `src/api/results.rs` with two endpoint handlers.

**Endpoint 1: GET /api/v1/results/:token**
Handler: `pub async fn get_results_by_token`
- Extract `token` from path parameter
- Call `db::scans::get_scan_by_token(pool, &token)` (from Plan 01)
- If None returned (not found or expired), return 404 with RFC 7807 body:
  ```json
  {
    "type": "https://trustedgeaudit.com/errors/results-not-found",
    "title": "Results Not Found",
    "status": 404,
    "detail": "Results not found or link has expired. Free scan results are available for 3 days."
  }
  ```
- If found, query findings via `db::findings::get_findings_by_scan(pool, scan.id)`
- Return JSON response matching the existing `get_scan` format but with additional fields:
  ```json
  {
    "id": "...",
    "target_url": "...",
    "status": "completed",
    "score": "B",
    "expires_at": "2026-02-08T...",
    "created_at": "...",
    "completed_at": "...",
    "findings": [...],
    "summary": { "total": N, "critical": N, "high": N, "medium": N, "low": N }
  }
  ```
- Do NOT include email address or submitter_ip in the response (privacy)
- Do NOT include scan_id in the response (prevent correlation to internal IDs)

**Endpoint 2: GET /api/v1/results/:token/download**
Handler: `pub async fn download_results_markdown`
- Same token lookup as above (404 if not found/expired)
- Generate a markdown report from the scan and findings:
  ```markdown
  # Security Scan Report

  **Target:** {url}
  **Grade:** {grade}
  **Scanned:** {completed_at formatted}
  **Report generated:** {now formatted}

  ---

  ## Summary

  | Severity | Count |
  |----------|-------|
  | Critical | {N} |
  | High | {N} |
  | Medium | {N} |
  | Low | {N} |
  | **Total** | **{N}** |

  ---

  ## Findings

  ### Critical

  #### 1. {title}

  **Severity:** Critical
  **Scanner:** {scanner_name}

  {description}

  **How to fix:**

  {remediation}

  ---

  (repeat for each severity level and finding)

  ---

  *Generated by TrustEdge Audit - https://trustedgeaudit.com*
  ```
- Return with headers:
  - `Content-Type: text/markdown; charset=utf-8`
  - `Content-Disposition: attachment; filename="trustedge-scan-{token_first_8_chars}.md"`

Add new routes to the Axum router:

```rust
    .allow_methods([Method::GET, Method::POST])
    .allow_headers(Any);

// Add to router:
.layer(cors)
```
