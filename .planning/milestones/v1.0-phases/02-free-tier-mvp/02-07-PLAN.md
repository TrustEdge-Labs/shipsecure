**Create `frontend/components/progress-checklist.tsx`** (client component):
Displays a checklist of scan stages with completion status.

```typescript
'use client'

interface ProgressChecklistProps {
  stages: {
    headers: boolean
    tls: boolean
    files: boolean
    secrets: boolean
  }
  status: string
}

export function ProgressChecklist({ stages, status }: ProgressChecklistProps) {
  const items = [
    { key: 'headers', label: 'Security Headers', done: stages.headers },
    { key: 'tls', label: 'TLS Configuration', done: stages.tls },
    { key: 'files', label: 'Exposed Files', done: stages.files },
    { key: 'secrets', label: 'JavaScript Secrets', done: stages.secrets },
  ]

  return (
    <ul className="space-y-3">
      {items.map(item => (
        <li key={item.key} className="flex items-center gap-3">
          {/* Completion indicator */}
          <span className={`flex-shrink-0 w-6 h-6 rounded-full flex items-center justify-center text-sm
            ${item.done
              ? 'bg-green-100 dark:bg-green-900 text-green-600 dark:text-green-400'
              : status === 'failed'
                ? 'bg-red-100 dark:bg-red-900 text-red-600 dark:text-red-400'
                : 'bg-gray-100 dark:bg-gray-800 text-gray-400 dark:text-gray-500 animate-pulse'
            }`}
          >
            {item.done ? '✓' : status === 'failed' ? '✗' : '○'}
          </span>
          <span className={item.done ? 'text-gray-900 dark:text-gray-100' : 'text-gray-500 dark:text-gray-400'}>
            {item.label}
          </span>
        </li>
      ))}
    </ul>
  )
}
```

**Create `frontend/app/scan/[id]/page.tsx`** (client component for polling):

This is a CLIENT component because it uses setInterval for polling.

```typescript
'use client'

import { useEffect, useState } from 'react'
import { useRouter, useParams } from 'next/navigation'
import { ProgressChecklist } from '@/components/progress-checklist'

interface ScanStatus {
  status: string
  results_token: string | null
  stage_headers: boolean
  stage_tls: boolean
  stage_files: boolean
  stage_secrets: boolean
  target_url: string
  error_message: string | null
}
```

Implementation:
1. Use `useParams()` to get the scan ID from the URL
2. Poll `GET /api/v1/scans/{id}` every 2 seconds using `setInterval`
3. Use `NEXT_PUBLIC_BACKEND_URL` (client-side env var) for the fetch URL
4. Display the target URL being scanned
5. Show the ProgressChecklist component with stage booleans from API response
6. When `status === 'completed'` AND `results_token` is present:
   - Clear the polling interval
   - Show "Scan complete!" message
   - Auto-redirect to `/results/{results_token}` after 1 second
7. When `status === 'failed'`:
   - Clear the polling interval
   - Show error message with "Try again" link back to landing page
8. Handle network errors: If 3 consecutive polls fail, show "Connection lost, retrying..." message (don't stop polling)
9. Clean up interval on component unmount

The page should also be bookmarkable — if a user returns to `/scan/{id}` after the scan completed, it should still show status and redirect if completed.

Page layout:
- Centered card on a clean background
- "Scanning {url}..." heading
- Progress checklist
- Subtle animated indicator (CSS pulse or spinner) while scanning
- No time estimate (per CONTEXT.md)
- Footer: "You can close this tab. We'll email you when your scan is ready."
