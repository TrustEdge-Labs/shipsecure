**A. src/db/scans.rs** -- Add soft-expire function and expired scan query:

1. Add `soft_expire_scans_by_tier` function (replaces `delete_expired_scans_by_tier` in cleanup):
   ```rust
   pub async fn soft_expire_scans_by_tier(pool: &PgPool, tier: &str) -> Result<u64, sqlx::Error> {
       let result = sqlx::query(
           "UPDATE scans
            SET status = 'expired'
            WHERE tier = $1
              AND status IN ('completed', 'failed')
              AND expires_at IS NOT NULL
              AND expires_at < NOW()"
       )
       .bind(tier)
       .execute(pool)
       .await?;
       Ok(result.rows_affected())
   }
   ```
   Note: No 24h grace period for soft expire -- mark expired as soon as expires_at passes. The old delete_expired_scans_by_tier had a 24h grace because deletion is destructive; soft-expire is not.

2. Add `get_scan_by_token_including_expired` function for results endpoint fallback:
   ```rust
   pub async fn get_scan_by_token_including_expired(pool: &PgPool, token: &str) -> Result<Option<Scan>, sqlx::Error> {
       let scan = sqlx::query_as::<_, Scan>(
           "SELECT id, target_url, email, submitter_ip::text, request_id, status, score, results_token,
                   expires_at::timestamp, detected_framework, detected_platform,
                   stage_headers, stage_tls, stage_files, stage_secrets, stage_detection, stage_vibecode,
                   tier, error_message, started_at::timestamp, completed_at::timestamp, created_at::timestamp,
                   clerk_user_id
            FROM scans
            WHERE results_token = $1"
       )
       .bind(token)
       .fetch_optional(pool)
       .await?;
       Ok(scan)
   }
   ```
   This query has NO expiry filter -- it returns the scan regardless of status, including expired.

3. Keep `delete_expired_scans_by_tier` for now (don't delete it -- other code may reference it for truly old data cleanup later).

**B. src/cleanup.rs** -- Switch from hard delete to soft expire:

Replace calls to `delete_expired_scans_by_tier` with `soft_expire_scans_by_tier` in `run_cleanup`:
```rust
async fn run_cleanup(pool: &PgPool) {
    let anon = match scans::soft_expire_scans_by_tier(pool, "free").await {
        Ok(n) => n,
        Err(e) => {
            tracing::error!(error = %e, "retention_cleanup_failed");
            return;
        }
    };

    let dev = match scans::soft_expire_scans_by_tier(pool, "authenticated").await {
        Ok(n) => n,
        Err(e) => {
            tracing::error!(error = %e, "retention_cleanup_failed");
            return;
        }
    };

    tracing::info!(
        anonymous_expired = anon,
        developer_expired = dev,
        total_expired = anon + dev,
        "retention_cleanup"
    );
}
```
Update log field names from `*_deleted` to `*_expired` for clarity.
