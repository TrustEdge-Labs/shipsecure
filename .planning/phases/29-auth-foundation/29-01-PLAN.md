1. **src/api/webhooks.rs** — Add `handle_clerk_webhook` function alongside the existing `handle_stripe_webhook`:
   ```rust
   pub async fn handle_clerk_webhook(
       State(state): State<AppState>,
       headers: HeaderMap,
       body: Bytes,
   ) -> Result<StatusCode, ApiError> {
   ```

   Implementation:
   a. Read `CLERK_WEBHOOK_SIGNING_SECRET` from env var. If missing, log error and return `ApiError::InternalError`.
   b. Create svix verifier: `let wh = svix::webhooks::Webhook::new(&secret)`. Map error to `ApiError::InternalError("Invalid webhook secret format")`.
   c. Convert Axum `HeaderMap` to `http::HeaderMap` — iterate headers and clone key/value pairs. This is necessary because svix expects `http::HeaderMap` and versions may differ. The conversion pattern:
      ```rust
      let mut http_headers = http::HeaderMap::new();
      for (name, value) in headers.iter() {
          if let Ok(name) = http::header::HeaderName::from_bytes(name.as_str().as_bytes()) {
              if let Ok(value) = http::header::HeaderValue::from_bytes(value.as_bytes()) {
                  http_headers.insert(name, value);
              }
          }
      }
      ```
   d. Call `wh.verify(&body, &http_headers)`. On failure: `tracing::warn!("Clerk webhook signature verification failed")` and return `ApiError::Unauthorized` (not ValidationError — this is an auth failure).
   e. Parse JSON body: `serde_json::from_slice::<serde_json::Value>(&body)`.
   f. Match on `event["type"]`:
      - `"user.created"`: Extract `clerk_user_id` from `event["data"]["id"]` and `email` from `event["data"]["email_addresses"][0]["email_address"]`. INSERT into users table: `INSERT INTO users (clerk_user_id, email) VALUES ($1, $2) ON CONFLICT (clerk_user_id) DO NOTHING`. Log with `tracing::info!(clerk_user_id = %clerk_user_id, "User created via Clerk webhook")`.
      - Default: `tracing::info!(event_type = %event_type, "Unhandled Clerk webhook event")`.
   g. Return `Ok(StatusCode::NO_CONTENT)` (204).

   Per user decision on webhook failure: log the error and return 500 — rely on Clerk/Svix automatic retry with exponential backoff. So on DB errors, return `ApiError::InternalError(...)` which maps to 500.
