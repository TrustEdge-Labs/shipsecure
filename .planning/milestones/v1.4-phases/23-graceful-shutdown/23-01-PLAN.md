<objective>
Add TaskTracker and CancellationToken to ScanOrchestrator, replacing fire-and-forget tokio::spawn with tracked spawns that respect shutdown signaling.

Purpose: Without task tracking, the Tokio runtime drops all in-flight scans immediately when main() returns. TaskTracker ensures scans can be waited on during graceful shutdown. CancellationToken prevents queued scans from starting after shutdown is initiated.

Output: Modified ScanOrchestrator with TaskTracker integration, shutdown-aware spawn methods, and a shutdown_gracefully() method for coordinated drain.
</objective>

**4. Replace `tokio::spawn` with `self.task_tracker.spawn` in `spawn_scan`:**
In the `spawn_scan` method, change `tokio::spawn(async move { ... })` to `self.task_tracker.spawn(async move { ... })`.

Add shutdown token checks at two points:
- **Before** incrementing scan_queue_depth (before semaphore acquire): If `shutdown_token.is_cancelled()`, return early with `tracing::info!("Shutdown in progress, skipping queued scan")` and do NOT increment queue depth gauge.
- **After** acquiring semaphore permit (after decrementing queue depth): If `shutdown_token.is_cancelled()`, return early with `tracing::info!("Shutdown in progress, aborting scan before execution")`. Do NOT increment active_scans gauge.

Clone `self.shutdown_token` into the async block alongside the other clones (pool, semaphore, etc.).

**5. Apply the same changes to `spawn_paid_scan`:**
Same pattern: replace `tokio::spawn` with `self.task_tracker.spawn`, add the same two shutdown checks (before queue depth increment, after semaphore acquire).

**6. Add shutdown method:**
```rust
/// Returns true if shutdown is in progress
pub fn is_shutting_down(&self) -> bool {
    self.shutdown_token.is_cancelled()
}

/// Get the CancellationToken for external use (middleware, health checks)
pub fn shutdown_token(&self) -> CancellationToken {
    self.shutdown_token.clone()
}

/// Initiate graceful shutdown: close tracker and cancel token
pub fn initiate_shutdown(&self) {
    self.task_tracker.close();
    self.shutdown_token.cancel();
}

/// Wait for all tracked tasks to complete
pub async fn wait_for_drain(&self) {
    self.task_tracker.wait().await;
}
```

**7. Also update `execute_scan` (the sync test method) if it uses tokio::spawn internally** -- it does not, so no changes needed there.

**Important:** The inner scanner spawns inside `run_scanners` (headers_handle, tls_handle, etc.) use `tokio::spawn` for parallelizing scanners WITHIN a single scan. These should NOT be converted to task_tracker.spawn because they are already joined within the scan task via `tokio::join!`. Only the top-level scan spawns need tracking.

In `src/orchestrator/mod.rs`, no changes needed -- it already re-exports ScanOrchestrator.
