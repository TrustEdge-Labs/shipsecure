**Create `frontend/app/opengraph-image.tsx`:**
- Use Next.js `ImageResponse` from `'next/og'` to generate a 1200x630 OG image.
- Export: `export const runtime = 'edge'`
- Export: `export const alt = 'ShipSecure - Security Scanning for Vibe-Coded Apps'`
- Export: `export const size = { width: 1200, height: 630 }`
- Export: `export const contentType = 'image/png'`
- Default export an async function `Image()` that returns `new ImageResponse(...)` with:
  - A container div styled with: `background: 'linear-gradient(135deg, #1e3a5f 0%, #2563eb 50%, #1e40af 100%)'`, full width/height, flexbox centered (both axes), `flexDirection: 'column'`, white text, `fontFamily: 'system-ui, sans-serif'`
  - Top element: a div with a shield icon (use Unicode U+1F6E1 or simple SVG-like text "[ ]" -- actually, since ImageResponse supports limited JSX, use text). Display "ShipSecure" in fontSize 80, fontWeight bold, marginBottom 20.
  - Subtitle: "Security Scanning for Vibe-Coded Apps" in fontSize 36, opacity 0.85
  - Bottom tagline: "Ship fast, stay safe." in fontSize 24, opacity 0.6, marginTop 40
- Keep it simple -- no custom fonts (stay under 500KB bundle limit). Use system fonts only.

**Create `frontend/app/sitemap.ts`:**
- Import `MetadataRoute` from `'next'`
- Default export a function returning `MetadataRoute.Sitemap` array with:
  - `{ url: 'https://shipsecure.ai', lastModified: new Date(), changeFrequency: 'daily', priority: 1.0 }`
- Do NOT include /results/*, /scan/*, /payment/*, or /api/* paths (private/transactional content).
- Note: Currently only one public page exists. More entries will be added when /privacy and /terms pages are created in Phase 10.

**Create `frontend/app/robots.ts`:**
- Import `MetadataRoute` from `'next'`
- Default export a function returning `MetadataRoute.Robots` with:
  - `rules`: `{ userAgent: '*', allow: '/', disallow: ['/results/', '/scan/', '/api/', '/payment/'] }`
  - `sitemap`: `'https://shipsecure.ai/sitemap.xml'`
- IMPORTANT: Do NOT block /results/ completely in robots.txt AND use noindex -- that would prevent crawlers from seeing the noindex tag. However, since results pages use generateMetadata with robots: { index: false }, and robots.txt Disallow is a suggestion not a mandate, having both is fine as defense-in-depth. Google respects noindex even when robots.txt blocks -- but Bing might not. This is acceptable for MVP.
