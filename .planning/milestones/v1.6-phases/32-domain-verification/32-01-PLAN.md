    ```sql
    CREATE TABLE IF NOT EXISTS verified_domains (
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        clerk_user_id TEXT NOT NULL REFERENCES users(clerk_user_id) ON DELETE CASCADE,
        domain TEXT NOT NULL,
        verification_token TEXT NOT NULL UNIQUE,
        status TEXT NOT NULL DEFAULT 'pending'
            CHECK (status IN ('pending', 'verified')),
        verified_at TIMESTAMPTZ,
        expires_at TIMESTAMPTZ,
        created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
        updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );
    CREATE UNIQUE INDEX idx_verified_domains_clerk_domain ON verified_domains(clerk_user_id, domain);
    CREATE INDEX idx_verified_domains_domain ON verified_domains(domain);
    ```
    Note: The `verification_token` column already has a UNIQUE constraint, which serves as an implicit index.

    **Model** (`src/models/domain.rs`):
    Create `VerifiedDomain` struct with `#[derive(Debug, Clone, sqlx::FromRow, Serialize)]`:
    - `id: uuid::Uuid`

    - `domain: String`
    - `verification_token: String`
    - `status: String`
    - `verified_at: Option<chrono::DateTime<chrono::Utc>>`

    **DB queries** (`src/db/domains.rs`):

    1. `get_verified_domain(pool, clerk_user_id, domain) -> Result<Option<VerifiedDomain>>` — SELECT by (clerk_user_id, domain).

    2. `upsert_pending_domain(pool, clerk_user_id, domain, token) -> Result<VerifiedDomain>` — INSERT ... ON CONFLICT (clerk_user_id, domain) DO UPDATE SET verification_token = EXCLUDED.verification_token, status = 'pending', verified_at = NULL, expires_at = NULL, updated_at = NOW() — RETURNING *. This handles re-verification of expired or pending domains. Note: the caller checks for already-verified (non-expired) domains BEFORE calling this; this function only runs for pending or expired records.

    3. `mark_verified(pool, clerk_user_id, domain, expires_at) -> Result<()>` — UPDATE verified_domains SET status = 'verified', verified_at = NOW(), expires_at = $3, updated_at = NOW() WHERE clerk_user_id = $1 AND domain = $2.

    4. `list_user_domains(pool, clerk_user_id) -> Result<Vec<VerifiedDomain>>` — SELECT * FROM verified_domains WHERE clerk_user_id = $1 ORDER BY created_at DESC.

    5. `is_domain_verified(pool, clerk_user_id, domain) -> Result<bool>` — SELECT EXISTS(SELECT 1 FROM verified_domains WHERE clerk_user_id = $1 AND domain = $2 AND status = 'verified' AND expires_at > NOW()). This is the performance-critical function called from results.rs on every authenticated result fetch.

    Use `sqlx::query()` non-macro pattern (consistent with Phase 29 pattern to avoid compile-time DB requirement). Use `sqlx::query_as::<_, VerifiedDomain>()` for queries returning full rows.

  <name>Task 2: Create domain API handlers with normalization, blocklist, SSRF protection, and meta tag verification</name>
  <files>

  <action>
    **API module** (`src/api/domains.rs`):

    **Domain normalization helper** — `normalize_domain(input: &str) -> Result<String, ApiError>`:
    - Trim input, handle `://` prefix by parsing as URL via `url::Url`, otherwise prepend `https://` and parse
    - Extract `host_str()`, strip `www.` prefix, lowercase
    - This function is used both internally and will be called from results.rs for consistent normalization

    **Shared-hosting TLD blocklist** — hardcoded `const BLOCKED_ROOT_TLDS: &[&str] = &["github.io", "vercel.app", "netlify.app", "pages.dev"];`
    - `is_blocked_root_tld(domain: &str) -> bool` — returns true ONLY for exact root TLD match (e.g., `vercel.app` blocked, `myapp.vercel.app` allowed). Per user decision: subdomains are allowed because vibe-coders deploy there.

    **Token generation** — `generate_verification_token() -> String`:
    - `rand::thread_rng().gen::<[u8; 32]>()` + `base64::engine::general_purpose::URL_SAFE_NO_PAD.encode()` (same pattern as `generate_results_token()` in worker_pool.rs). 256-bit entropy, opaque.

    **Meta tag verification** — `fetch_and_check_meta_tag(target_url: &str, expected_token: &str) -> Result<(), VerificationFailureReason>`:
    - Build reqwest client with 15s timeout, limited redirects (5), UA "ShipSecure-Verifier/1.0"
    - Fetch the URL, parse HTML with `scraper::Html::parse_document()`
    - Use `Selector::parse("meta[name='shipsecure-verification']")` to find the meta tag
    - If not found, return `TagNotFound`
    - If found but content doesn't match expected token, return `WrongContent`
    - If found and matches, return Ok(())
    - Also check `head meta[name='shipsecure-verification']` vs `body meta[name='shipsecure-verification']` for diagnostic info (but verification succeeds either way — per research Pitfall 3, proving HTML control is the goal)
    - Return a `VerificationFailureReason` enum: `FetchFailed(String)`, `TagNotFound`, `TagInBody`, `WrongContent`

    **Handler: `verify_start`** — `POST /api/v1/domains/verify-start`:
    - Mandatory auth: `Claims(claims): Claims<ClerkClaims>` extractor (401 automatic for unauthenticated)
    - Parse `Json<VerifyStartRequest>` with `domain: String`
    - Normalize domain
    - Check blocklist — return 400 with descriptive error: "'{domain}' is a shared hosting platform. Enter your app's subdomain instead (e.g., myapp.{domain})."
    - Check for already-verified non-expired domain via `db::domains::get_verified_domain()` — if found and verified with expires_at > NOW(), return 200 with `already_verified: true, domain, expires_in_days`
    - Generate token, build meta_tag string: `<meta name="shipsecure-verification" content="{token}">`
    - Upsert pending domain via `db::domains::upsert_pending_domain()`
    - Return 201 with `{ domain, token, meta_tag }`

    **Handler: `verify_confirm`** — `POST /api/v1/domains/verify-confirm`:
    - Mandatory auth via `Claims<ClerkClaims>`
    - Parse `Json<VerifyConfirmRequest>` with `domain: String`
    - Normalize domain
    - Look up pending verification record — 400 if not found ("No verification started for this domain")
    - Build target URL: `https://{domain}`
    - **SSRF validation** — call `crate::ssrf::validate_scan_target(&target_url).await` BEFORE any outbound fetch. Return error if SSRF blocked.
    - Call `fetch_and_check_meta_tag()` with stored token
    - On success: `db::domains::mark_verified()` with `expires_at = Utc::now() + Duration::days(30)`, return `{ verified: true, domain, expires_at }`
    - On failure: return `{ verified: false, domain, failure_reason: "..." }` with specific diagnosis messages per VerificationFailureReason:
      - FetchFailed: "We couldn't reach your site: {e}. Make sure it's publicly accessible."
      - TagNotFound: "We fetched your page but didn't find the meta tag. Check that it's in <head>, not <body>."
      - TagInBody: "We found the meta tag in <body>, not <head>. Move it to the <head> section for best results. Verification still succeeded." (note: this is a soft warning, verification passes)
      - WrongContent: "We found the meta tag but the content doesn't match the expected token. Copy the snippet again and replace any existing tag."

    **Handler: `verify_check`** — `POST /api/v1/domains/verify-check` ("Test my tag" pre-check):
    - Mandatory auth via `Claims<ClerkClaims>`
    - Same normalization, lookup, and SSRF validation as verify_confirm
    - Call `fetch_and_check_meta_tag()` — returns same diagnostic messages
    - Does NOT write to DB (no `mark_verified` call) — this is the pre-check only
    - Returns `{ found: true/false, domain, message: "..." }` with same diagnosis messages

    **Handler: `list_domains`** — `GET /api/v1/domains`:
    - Mandatory auth via `Claims<ClerkClaims>`
    - Call `db::domains::list_user_domains(pool, clerk_user_id)`
    - Return JSON array of VerifiedDomain records

    **Module registration**: Add `pub mod domains;` to `src/api/mod.rs`.

    ```rust
    .route("/api/v1/domains/verify-check", post(domains::verify_check))
    .route("/api/v1/domains", get(domains::list_domains))
    ```
    Import `domains` from `crate::api::domains`. Use existing `post` and `get` from `axum::routing`.

    **Critical change:** The current `owner_verified` logic in `get_results_by_token` and `download_results_markdown` (from Phase 31) only checks identity match:
    ```rust
    let owner_verified = match (&authenticated_user_id, &scan.clerk_user_id) {
        (Some(caller), Some(owner)) => caller == owner,
        _ => false,
    };
    ```

    **Extend to also check domain verification:**
    1. When identity matches (`caller == owner`), extract the domain from `scan.target_url` using the `normalize_domain` helper from `domains.rs` (make it `pub` in domains.rs or create a shared helper). Use `url::Url::parse()` to get the host, then strip `www.` prefix — same normalization as verify-start.
    2. Call `db::domains::is_domain_verified(&state.pool, caller, &domain).await.unwrap_or(false)`
    3. `owner_verified = identity_match && domain_verified`

    This means:
    - If user is the scan owner AND has active (non-expired) domain verification → `owner_verified: true` → full results
    - If user is the scan owner but domain verification expired → `owner_verified: false` → high/critical gated
    - If user is not the scan owner → `owner_verified: false` → high/critical gated
    - Anonymous → `owner_verified: false` → high/critical gated

    **Apply to both handlers:** `get_results_by_token` and `download_results_markdown` — they both have the same `owner_verified` computation block. Update both consistently.

    **Extract domain helper function:** Create a helper in results.rs (or import from domains.rs):
    ```rust
    fn extract_domain_from_url(target_url: &str) -> Option<String> {
        let parsed = url::Url::parse(target_url).ok()?;
        let host = parsed.host_str()?;
        let domain = host.strip_prefix("www.").unwrap_or(host);
        Some(domain.to_lowercase())
    }
    ```
    This must match the normalization in `domains.rs` — lowercase, no www prefix — to avoid Pitfall 6 (normalization mismatch).
