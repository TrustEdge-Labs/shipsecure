  <action>
1. **In `src/main.rs`**, add a `validate_required_env_vars()` function:

   ```rust
   fn validate_required_env_vars(vars: &[&str]) -> Result<(), String> {
       let mut missing = Vec::new();
       for var in vars {
           if std::env::var(var).is_err() {
               missing.push(*var);
           }
       }
       if !missing.is_empty() {
           return Err(format!(
               "Missing required environment variables:\n  - {}\n\nSee .env.example for configuration.",
               missing.join("\n  - ")
           ));
       }
       Ok(())
   }
   ```

2. **Call validation immediately after `dotenvy::dotenv().ok()`** (before tracing init, before anything else):

   ```rust
   validate_required_env_vars(&[
       "DATABASE_URL",
       "PORT",
       "RUST_LOG",
       "TRUSTEDGE_BASE_URL",
       "FRONTEND_URL",
       "MAX_CONCURRENT_SCANS",
   ]).expect("Configuration error");
   ```

   Note: RESEND_API_KEY, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, NUCLEI_BINARY_PATH, TESTSSL_BINARY_PATH are intentionally NOT in the required list -- they are optional (scanner/service graceful degradation).

3. **Remove the default for PORT** -- change from:
   ```rust
   let port: u16 = std::env::var("PORT")
       .unwrap_or_else(|_| "3000".to_string())
       .parse()
       .expect("PORT must be a number");
   ```
   To:
   ```rust
   let port: u16 = std::env::var("PORT")
       .expect("PORT must be set")
       .parse()
       .expect("PORT must be a valid number");
   ```
   (PORT is now validated as required above, so the expect will never fire, but keeps the type-parsing error clear.)

4. **Parse MAX_CONCURRENT_SCANS from env var** and pass to orchestrator:
   ```rust
   let max_concurrent: usize = std::env::var("MAX_CONCURRENT_SCANS")
       .expect("MAX_CONCURRENT_SCANS must be set")
       .parse()
       .expect("MAX_CONCURRENT_SCANS must be a valid number");
   ```
   Change `ScanOrchestrator::new(pool.clone(), 5)` to `ScanOrchestrator::new(pool.clone(), max_concurrent)`.

5. **Remove the default for TRUSTEDGE_BASE_URL** in `src/orchestrator/worker_pool.rs`:
   Change:
   ```rust
   let base_url = std::env::var("TRUSTEDGE_BASE_URL")
       .unwrap_or_else(|_| "http://localhost:3001".to_string());
   ```
   To:
   ```rust
   let base_url = std::env::var("TRUSTEDGE_BASE_URL")
       .expect("TRUSTEDGE_BASE_URL must be set");
   ```
   (Already validated as required at startup, so this is safe.)

6. **Remove the default for RUST_LOG** in tracing init. Change:
   ```rust
   EnvFilter::try_from_default_env()
       .unwrap_or_else(|_| EnvFilter::new("info"))
   ```
   To:
   ```rust
   EnvFilter::try_from_default_env()
       .expect("RUST_LOG must be set to a valid filter (e.g., info,trustedge_audit=debug)")
   ```
   (RUST_LOG is validated as required at startup.)

  <action>
Replace the current 3-line `.env.example` with a comprehensive, grouped, documented template. Include ALL environment variables the application reads (from code audit), organized into logical sections.

**Required variables** (application crashes without these):
- DATABASE_URL
- PORT
- RUST_LOG
- TRUSTEDGE_BASE_URL
- FRONTEND_URL
- MAX_CONCURRENT_SCANS

**Optional variables** (features gracefully disabled without these):
- RESEND_API_KEY (email delivery disabled)
- STRIPE_SECRET_KEY (checkout disabled)
- STRIPE_WEBHOOK_SECRET (webhook validation disabled)
- NUCLEI_BINARY_PATH (Nuclei scans skipped)
- TESTSSL_BINARY_PATH (testssl scans skipped)
- TRUSTEDGE_TEMPLATES_DIR (defaults to templates/nuclei/)

Use this format for each group:
```
# ========================================
# GROUP NAME
# ========================================

# Description of variable (REQUIRED)
# Format/options/example
VARIABLE_NAME=placeholder_value
```

For optional variables, comment out the value line:
```
# Description (OPTIONAL - feature disabled if not set)
# Get from: https://...
# VARIABLE_NAME=placeholder
```

Development-ready defaults in placeholder values (e.g., DATABASE_URL points to localhost postgres with dev credentials).

Do NOT include any real secrets or API keys.
