<task type="auto">
  <name>Task 1: Install Playwright, configure for production build with testProxy</name>
  <files>
    frontend/package.json
    frontend/playwright.config.ts
    frontend/next.config.ts
  </files>
  <action>
1. Install Playwright as dev dependency:
   ```
   cd frontend && npm install -D @playwright/test && npx playwright install --with-deps chromium
   ```

2. Create `frontend/playwright.config.ts` importing `defineConfig` from `next/experimental/testmode/playwright` (NOT from `@playwright/test`). Configuration:
   - `testDir: './e2e'`
   - `fullyParallel: false`
   - `forbidOnly: !!process.env.CI`
   - `retries: process.env.CI ? 1 : 0`
   - `workers: 1` (single worker to avoid port conflicts)
   - `reporter: process.env.CI ? 'dot' : 'html'`
   - `use.baseURL: 'http://localhost:3000'`
   - `use.trace: 'on-first-retry'`
   - Single project: `name: 'chromium'`, `viewport: { width: 1280, height: 720 }`
   - `webServer.command: 'npm run start'`
   - `webServer.url: 'http://localhost:3000'`
   - `webServer.reuseExistingServer: !process.env.CI`
   - `webServer.timeout: 120_000`
   - `webServer.stdout: 'ignore'`
   - `webServer.stderr: 'pipe'`
   - `webServer.env: { PLAYWRIGHT_TEST: '1' }` — this activates testProxy in next.config.ts

3. Update `frontend/next.config.ts` to add `experimental.testProxy`:
   ```typescript
   const nextConfig: NextConfig = {
     output: 'standalone',
     experimental: {
       testProxy: process.env.PLAYWRIGHT_TEST === '1',
     },
   };
   ```
   This is conditionally enabled so production/dev builds are not affected.

4. Update `frontend/package.json` scripts:
   - Change `test:e2e` from the placeholder echo to: `"test:e2e": "playwright test"`
   - Add `"test:e2e:ui": "playwright test --ui"` for interactive debugging

Note: The webServer.command uses `npm run start` which runs `next start`. The production build (`npm run build`) must be run BEFORE `npm run test:e2e`. This is expected — CI will handle the build step (Phase 28), and locally developers run `npm run build` first.
  </action>
  <verify>
Run `cd frontend && npx playwright --version` to confirm Playwright is installed. Run `npx playwright test --help` to confirm config is parseable. Verify `next.config.ts` contains `testProxy`. Verify `package.json` has updated `test:e2e` script.
  </verify>
  <done>
Playwright installed with chromium browser. Config file exists at `frontend/playwright.config.ts` with production webServer, single worker, testProxy env var. `next.config.ts` conditionally enables testProxy. `package.json` has working `test:e2e` script.
  </done>
</task>

  <action>
Create E2E-specific fixtures (separate from MSW fixtures in `__tests__/helpers/fixtures/`). These must match the real API response shapes exactly (per locked decision).

**fixtures/scan.ts** — Export `scanFixtures` object with:
- `created`: `{ id: 'scan_e2e_001', status: 'pending', url: '/api/v1/scans/scan_e2e_001', target_url: 'https://example.com', results_token: null, stage_detection: false, stage_headers: false, stage_tls: false, stage_files: false, stage_secrets: false, stage_vibecode: false, error_message: null }` (matches CreateScanResponse + ScanStatus shape)
- `inProgress`: Same id, `status: 'in_progress'`, some stages true (`stage_detection: true, stage_headers: true`), `results_token: null`
- `completed`: Same id, `status: 'completed'`, all stages true, `results_token: 'tok_e2e_abc123'`
- `failed`: Same id, `status: 'failed'`, `error_message: 'Target website is unreachable'`

**fixtures/results.ts** — Export `resultsFixtures` object with:
- `freeGradeB`: Full ScanResponse shape with `tier: 'free'`, `score: 'B'`, `status: 'completed'`, findings array with 3 findings (1 high, 1 medium, 1 low severity), summary object, detected_framework/platform, expires_at, target_url, etc. This must include ALL fields from the ScanResponse type.
- `paidGradeA`: Full ScanResponse shape with `tier: 'paid'`, `score: 'A'`, findings array with 1 low finding, summary, detected_framework: 'Next.js', detected_platform: 'Vercel'.

**fixtures/checkout.ts** — Export `checkoutFixtures` object with:
- `success`: `{ checkout_url: 'https://checkout.stripe.com/c/pay/cs_test_e2e_mock123' }`
- `error`: `{ type: 'about:blank', title: 'Failed to create checkout session', status: 422, detail: 'Invalid scan ID' }`

**helpers/route-mocks.ts** — Reusable `page.route()` interceptors. Export functions:
- `mockScanPolling(page, fixtures)`: Sets up stateful polling interceptor on `**/api/v1/scans/**`. Returns `inProgress` for first N calls (using closure counter), then `completed`. Adds 200ms delay per locked decision (100-500ms range). Accepts the scan fixtures as parameter.
- `mockCheckout(page, checkoutUrl)`: Intercepts `**/api/v1/checkout` POST, returns checkout URL fixture with 150ms delay. Also intercepts `https://checkout.stripe.com/**` with 302 redirect to success page.
- `mockNetworkFailure(page, urlPattern)`: Calls `route.abort('failed')` on the given URL pattern.

**helpers/fetch-mocks.ts** — Reusable `next.onFetch()` interceptors. Export functions:
- `mockScanSubmission(next, scanFixtures)`: Intercepts POST to `/api/v1/scans` server-side (the Server Action's outbound fetch). Returns `201` with `scanFixtures.created`. Adds 200ms delay.
- `mockResultsPage(next, resultsFixture)`: Intercepts GET to `/api/v1/results/` server-side. Returns `200` with the provided results fixture. Adds 150ms delay.
- `mockScanCount(next, count)`: Intercepts GET to `/api/v1/stats/scan-count`. Returns `{ count }`. This is needed because the home page fetches scan count server-side.
- `mockResultsNotFound(next)`: Intercepts GET to `/api/v1/results/` and returns 404.
- `mockServerError(next)`: Intercepts any backend URL and returns 500.

All delay values should use `await new Promise(r => setTimeout(r, delayMs))` where delayMs is in the 100-500ms range per locked decision.

The `next` parameter in fetch-mocks has type from `next/experimental/testmode/playwright` — use the type annotation from the test fixture parameter.
