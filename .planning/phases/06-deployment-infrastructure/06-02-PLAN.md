**infrastructure/tasks/security.yml** — SSH hardening, deploy user, UFW firewall:

1. **Create deploy user** — Use `user` module:
   - `name: "{{ deploy_user }}"` (deploy)
   - `groups: sudo`
   - `append: yes`
   - `create_home: yes`
   - `shell: /bin/bash`
   - `state: present`

2. **Add SSH authorized key for deploy user** — Use `authorized_key` module:
   - `user: "{{ deploy_user }}"`
   - `key: "{{ deploy_user_ssh_key }}"`

3. **Harden SSH configuration** — Use `lineinfile` module with `validate: 'sshd -t -f %s'`:
   - Set `Port {{ ssh_port }}` (2222)
   - Set `PermitRootLogin no`
   - Set `PasswordAuthentication no`
   - Set `PubkeyAuthentication yes`
   - Set `MaxAuthTries 3`
   - Set `LoginGraceTime 20`
   - Use `notify: Restart SSH` handler

4. **Configure UFW firewall rules** — Use `community.general.ufw` module:
   - FIRST: Allow SSH on `{{ ssh_port }}` (CRITICAL: must be before enabling UFW to avoid lockout)
   - Allow port 80/tcp (HTTP)
   - Allow port 443/tcp (HTTPS)
   - Set default incoming policy: deny
   - Set default outgoing policy: allow
   - Enable UFW: `state: enabled`

5. **Define handlers section at the end**:
   - Handler `Restart SSH`: restart sshd service

**CRITICAL SAFETY:** The UFW SSH allow rule MUST come before `state: enabled`. Research Pitfall 1 warns about lockout from SSH changes. The handler restarts SSH after ALL tasks complete (not immediately), giving Ansible time to finish.

After SSH hardening, subsequent plays will connect as deploy user on port 2222. The main playbook's Phase 2 play initially connects as root on port 22. The SSH port change takes effect after the handler fires (end of play). A follow-up connection will use the new port. To handle this properly, add a task at the end that updates the in-memory inventory to use the new SSH port and deploy user for subsequent plays.

**infrastructure/tasks/docker.yml** — Docker CE installation:

1. **Install Docker prerequisites** — Use `apt` module:
   - `apt-transport-https`, `ca-certificates`, `curl`, `gnupg`, `lsb-release`
   - `update_cache: yes`

2. **Add Docker GPG key** — Use `ansible.builtin.apt_key`:
   - `url: https://download.docker.com/linux/ubuntu/gpg`

3. **Add Docker APT repository** — Use `apt_repository`:
   - `repo: "deb [arch=amd64] https://download.docker.com/linux/ubuntu {{ ansible_distribution_release }} stable"`

4. **Install Docker packages** — Use `apt`:
   - `docker-ce`, `docker-ce-cli`, `containerd.io`, `docker-compose-plugin`
   - `update_cache: yes`

5. **Add deploy user to docker group** — Use `user`:
   - `name: "{{ deploy_user }}"`
   - `groups: docker`
   - `append: yes`

6. **Enable and start Docker service** — Use `systemd`:
   - `name: docker`
   - `enabled: yes`
   - `state: started`

7. **Verify Docker installation** — Use `command: docker --version`:
   - `changed_when: false`
   - Register result and debug print version
