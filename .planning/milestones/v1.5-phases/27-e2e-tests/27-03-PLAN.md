**Test 5: "network timeout shows connection error"**

Per locked decision: simulate API not responding, verify timeout/connection error message.

Steps:
1. Navigate to scan progress page: `await page.goto('/scan/scan_e2e_timeout')`
2. Mock the polling endpoint to abort: `await page.route('**/api/v1/scans/scan_e2e_timeout', route => route.abort('failed'))`
3. The scan page has an `errorCount` state that increments on fetch failures. After 3+ errors, it shows a warning: "Having trouble connecting to our servers"
4. Wait for the error count to build up (the page polls every 2 seconds, so wait ~8-10 seconds): `await expect(page.locator('text=Having trouble connecting')).toBeVisible({ timeout: 15000 })`
5. **Recovery:** Verify the page still shows the scan UI (not crashed) and the user can navigate away
