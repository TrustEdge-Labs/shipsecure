**Create `frontend/app/payment/success/layout.tsx`:**
The payment success page (`page.tsx`) is a `'use client'` component because it calls `useEffect` for Plausible tracking. Client components CANNOT export `metadata` -- the build will fail. The solution is to create a server-side `layout.tsx` in the same directory that exports the metadata.

Create a new file `frontend/app/payment/success/layout.tsx` with:
```typescript
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Payment Successful - ShipSecure',
  description: 'Your deep security audit is processing. You will receive a PDF report by email.',
  robots: {
    index: false,
    follow: true,
  },
}

export default function PaymentSuccessLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return children
}
```

Notes:
- `index: false` because this is a transactional page with no SEO value
- `follow: true` because the page links back to the homepage (let crawlers follow that link)
- The layout simply renders `children` (the existing page.tsx) -- it's a metadata-only wrapper
- Title is 35 chars (under 60 limit), description is 82 chars (under 155 limit)
- Do NOT modify the existing `page.tsx` -- it keeps its `'use client'` directive and Plausible tracking logic
