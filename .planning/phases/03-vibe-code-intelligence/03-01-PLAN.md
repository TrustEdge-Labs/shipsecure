1. Add `pub mod detector;` to `src/scanners/mod.rs`.

2. Create `src/scanners/detector.rs` implementing the detection engine. This module:

   a. Define `const HIGH_CONFIDENCE_THRESHOLD: u8 = 60;`

   b. Implement `pub async fn detect_stack(target_url: &str) -> Result<DetectionResult, ScannerError>`:
      - Make a GET request to target_url using reqwest (with 15-second timeout, follow redirects)
      - Capture both response headers and body HTML
      - Run `detect_framework(headers, html)` and `detect_platform(headers)`
      - Return DetectionResult with signals list

   c. Implement `fn detect_framework(headers: &reqwest::header::HeaderMap, html: &str) -> (Option<Framework>, u8, Vec<String>)`:
      Uses weighted scoring per framework. Check all 4 frameworks and return the one with highest score above threshold.

      **Next.js signals (use scraper crate for HTML parsing):**
      - `#__NEXT_DATA__` script tag with valid JSON containing "buildId" (40 points, STRONG)
      - `script[src]` containing "/_next/static" (30 points, MEDIUM)
      - `x-powered-by` header containing "Next.js" (20 points, LOW - often stripped)
      - `meta[name='generator']` containing "Next.js" (10 points, LOW)

      **Vite/React signals:**
      - Script src containing "/.vite/" or script with `type="module"` + src containing hash pattern typical of Vite (30 points)
      - HTML body containing `<div id="root"></div>` or `<div id="app"></div>` (20 points - React mount point)
      - `import.meta` references in inline scripts (20 points - Vite-specific)
      - Presence of React indicators (data-reactroot, __REACT) WITHOUT Next.js signals (10 points)
      - CRITICAL: Only score Vite/React if Next.js score < threshold (Next.js uses React internally)

      **SvelteKit signals:**
      - Script src containing "/__sveltekit/" or "/_app/" with Svelte patterns (40 points, STRONG)
      - HTML containing `data-sveltekit` attributes (30 points, MEDIUM)
      - `meta[name='generator']` containing "SvelteKit" (10 points, LOW)

      **Nuxt signals:**
      - `#__NUXT__` or `#__NUXT_DATA__` script tag (40 points, STRONG)
      - Script src containing "/_nuxt/" (30 points, MEDIUM)
      - `x-powered-by` header containing "Nuxt" (20 points, LOW)

      When multiple frameworks score above threshold, pick the highest. If tie, prefer the one with strongest individual signal.

   d. Implement `fn detect_platform(headers: &reqwest::header::HeaderMap) -> (Option<Platform>, u8, Vec<String>)`:
      Platform detection is definitive (headers are strong signals):
      - `x-vercel-id` header present -> Platform::Vercel (confidence 100)
      - `x-nf-request-id` header present -> Platform::Netlify (confidence 100)
      - `x-railway-request-id` header present -> Platform::Railway (confidence 100)
      - Fallback: `server` header containing "Vercel"/"Netlify" (confidence 80)
      - No match -> None

   e. Add unit tests:
      - Test Next.js detection with mock HTML containing __NEXT_DATA__ + _next/static (should detect)
      - Test Next.js detection with only __NEXT_DATA__ (score 40, below threshold, should NOT detect)
      - Test Vercel platform detection from x-vercel-id header
      - Test no detection when HTML has no framework signals
      - Test Vite/React is NOT detected when Next.js signals are present (disambiguation)

Use `ScannerError` from existing `security_headers` module (or create a shared one in detector.rs). The function signature should match the pattern: `async fn -> Result<T, ScannerError>`.

Note: For partial matches per Claude's Discretion, if React is detected but no meta-framework (Next.js/Vite), do NOT report a framework. The user requirement is "Vite/React" meaning "React built with Vite" specifically.
