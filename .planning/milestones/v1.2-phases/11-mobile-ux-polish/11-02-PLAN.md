    Create Next.js App Router loading and error files at root and results route levels.

    **frontend/app/loading.tsx (Root loading fallback):**
    - Simple centered spinner with "Loading..." text.
    - Use `animate-spin` spinner (same style as existing scan progress page: `rounded-full h-8 w-8 border-b-2 border-blue-600`).
    - Wrap in a centered flex container: `flex items-center justify-center min-h-[50vh]`.
    - Must NOT use `'use client'` -- loading.tsx is a Server Component by default.

    **frontend/app/results/[token]/loading.tsx (Results skeleton):**
    - Must match the actual results page layout to prevent CLS (Cumulative Layout Shift).
    - Use `animate-pulse` Tailwind class on placeholder divs.
    - Structure: header card skeleton (h-8 rounded bar for title, h-4 bars for metadata), grade summary skeleton (h-16 rounded block), findings section skeleton (3x h-24 rounded blocks for finding accordions), actions skeleton (2x h-10 rounded bars for buttons).
    - Use same container: `min-h-screen bg-gray-50 dark:bg-gray-950 py-8 px-4` and `max-w-4xl mx-auto`.
    - Must NOT use `'use client'`.

    **frontend/app/error.tsx (Root error boundary):**
    - Must have `'use client'` directive at top.
    - Accept `{ error, reset }` props per Next.js convention.
    - Log error with `console.error` in a `useEffect`.
    - Display a centered card (same styling as scan progress error states): icon, title "Something went wrong", descriptive message "An unexpected error occurred. Please try again or return to the homepage.", two buttons: "Try again" (calls `reset()`) and "Return to Home" (links to `/`).
    - Use Tailwind dark mode classes consistent with existing pages.
    - Buttons must have `min-h-[44px]` for touch targets.

    **frontend/app/global-error.tsx (Catch-all for root layout errors):**
    - Must have `'use client'` directive.
    - Must render its own `<html>` and `<body>` tags (this replaces the root layout when triggered).
    - Simple error display with "Something went wrong" and a "Try again" button that calls `reset()`.
    - Minimal styling (inline or basic Tailwind classes -- this is the last-resort fallback).

    **frontend/app/results/[token]/error.tsx (Results-specific error boundary):**
    - Must have `'use client'` directive.
    - Accept `{ error, reset }` props.
    - Log error with `console.error` in `useEffect`.
    - Parse error context to provide specific guidance:
      - Default message: "Unable to load scan results. The results may have expired or the link may be invalid."
      - Suggested actions: "Try again" button (reset), "Start new scan" link to `/`.
    - Use same centered card pattern as other error states in the app.
    - Use responsive layout: `container mx-auto px-4 py-8 max-w-2xl`.
