**Step 1: Modify `frontend/app/actions/scan.ts`**

Read the existing file first. Then:

1. Add `authorization` field to the Zod `scanSchema`:
   ```typescript
   authorization: z
     .string()
     .transform(val => val === 'on')
     .refine(val => val === true, {
       message: 'You must confirm you have authorization to scan this website'
     })
   ```
   Note: HTML checkboxes send "on" when checked, and are absent from FormData when unchecked.

2. Update `ScanFormState` interface to include authorization in errors:
   ```typescript
   export interface ScanFormState {
     errors?: {
       url?: string[]
       email?: string[]
       authorization?: string[]
       _form?: string[]
     }
     scanId?: string
   }
   ```

3. Update `submitScan` to include authorization in the safeParse call:
   ```typescript
   const validatedFields = scanSchema.safeParse({
     url: formData.get('url'),
     email: formData.get('email'),
     authorization: formData.get('authorization') ?? '',
   })
   ```
   Note: Use `?? ''` because unchecked checkboxes return null from FormData.get(), which would cause a Zod type error instead of the custom refine message.

4. Do NOT change the fetch call to the backend -- the authorization field is frontend-only consent, not sent to the Rust backend.
