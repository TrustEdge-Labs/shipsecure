Create the SSRF protection module that validates target URLs before scanning.

**src/ssrf/mod.rs:**


**src/ssrf/validator.rs:**
Implement `validate_scan_target(url: &str) -> Result<String, SsrfError>` that:

1. Parse the URL (use url::Url crate -- add `url = "2"` to Cargo.toml if not present). Reject URLs without http/https scheme. Reject URLs with no host.

2. Extract the host. Reject if host is an IP literal that is private/loopback/link-local/metadata (skip DNS resolution for raw IPs).

3. For hostname strings, resolve DNS using `tokio::net::lookup_host` (async DNS resolution). Collect ALL resolved IP addresses.

4. Check EVERY resolved IP against the blocklist:
   - IPv4: `is_private()`, `is_loopback()`, `is_link_local()`, `is_unspecified()`, `is_multicast()`
   - IPv4: explicit check for `169.254.169.254` (AWS/GCP metadata)
   - IPv4: explicit check for `100.100.100.200` (Alibaba metadata)
   - IPv6: `is_loopback()`, `is_unspecified()`, `is_multicast()`
   - IPv6: check `fd00:ec2::254` prefix (AWS IPv6 metadata)
   - Reject if ANY resolved IP is blocked

5. On success, return the validated URL string (the original URL, not the resolved IP -- DNS rebinding protection is deferred per CONTEXT.md).

Define `SsrfError` enum:
- InvalidUrl(String)
- PrivateIp
- LoopbackIp
- LinkLocalIp
- CloudMetadata
- DnsResolutionFailed(String)
- BlockedScheme

Implement `std::fmt::Display` and `std::error::Error` for SsrfError.

**Update src/lib.rs** to add `pub mod ssrf;`

IMPORTANT: Use `tokio::net::lookup_host` (async), NOT `std::net::ToSocketAddrs` (blocking). This is an async application and blocking DNS resolution in an async context will starve the Tokio runtime.

Note: DNS rebinding protection (resolving once and using the resolved IP for the HTTP request) is explicitly deferred per CONTEXT.md decisions. Phase 1 resolves and validates, then passes the original URL to reqwest (which re-resolves). This is a known trade-off accepted by the user.

**src/scanners/mod.rs:**

**src/scanners/security_headers.rs:**
Implement `pub async fn scan_security_headers(url: &str) -> Result<Vec<Finding>, ScannerError>` that:

1. Create a reqwest::Client with 30-second timeout, no redirect following (check the LANDING page headers, not the redirect target). Set a custom User-Agent: "TrustEdge-Scanner/0.1".

2. Send a GET request to the URL. On HTTP error, return ScannerError.

3. Check for these 6 security headers and produce findings:

   a. **Content-Security-Policy (CSP):**
      - Missing: Severity::High, title "Missing Content-Security-Policy header"
      - Description: "Your site does not set a Content-Security-Policy header. This leaves it vulnerable to cross-site scripting (XSS) attacks because browsers will execute any inline scripts or load resources from any source."
      - Remediation: "Add a Content-Security-Policy header to your server response. Start with a restrictive policy like `Content-Security-Policy: default-src 'self'; script-src 'self'` and gradually relax it as needed."

   b. **Strict-Transport-Security (HSTS):**
      - Missing: Severity::High, title "Missing Strict-Transport-Security header"
      - Description: "Your site does not enforce HTTPS via the HSTS header. Users who type your domain without 'https://' could be intercepted by a man-in-the-middle attack during the initial HTTP request."
      - Remediation: "Add `Strict-Transport-Security: max-age=31536000; includeSubDomains` to all HTTPS responses. Ensure your site works fully over HTTPS first."

   c. **X-Frame-Options:**
      - Missing: Severity::Medium, title "Missing X-Frame-Options header"
      - Description: "Your site can be embedded in iframes on other domains, making it vulnerable to clickjacking attacks where users are tricked into clicking hidden elements."
      - Remediation: "Add `X-Frame-Options: DENY` (or SAMEORIGIN if you need iframes on your own domain) to your server responses."

   d. **X-Content-Type-Options:**
      - Missing: Severity::Medium, title "Missing X-Content-Type-Options header"
      - Description: "Without this header, browsers may try to guess the content type of your responses, which can lead to MIME-sniffing attacks where scripts are executed from non-script content types."
      - Remediation: "Add `X-Content-Type-Options: nosniff` to your server responses."

   e. **Referrer-Policy:**
      - Missing: Severity::Low, title "Missing Referrer-Policy header"
      - Description: "Your site does not control how much referrer information is shared when users click links to other sites. This could leak sensitive URL paths or query parameters to third parties."
      - Remediation: "Add `Referrer-Policy: strict-origin-when-cross-origin` to your server responses."

   f. **Permissions-Policy:**
      - Missing: Severity::Low, title "Missing Permissions-Policy header"
      - Description: "Your site does not restrict which browser features (camera, microphone, geolocation) can be used. Malicious scripts could potentially access these features."
      - Remediation: "Add `Permissions-Policy: camera=(), microphone=(), geolocation=()` to disable features you don't need."

4. For each header that IS present, do NOT create a finding (only report problems, not successes -- keeps results focused).

5. Set `scanner_name = "security_headers"` on all findings. Set `raw_evidence` to the actual response headers as a formatted string (for debugging).

6. Generate UUIDs for finding IDs using `Uuid::new_v4()`. Set `scan_id` to Uuid::nil() as a placeholder (the orchestrator will set the real scan_id when persisting).

Define `ScannerError` enum: HttpError(String), Timeout, Other(String). Implement Display and Error.

**src/scanners/aggregator.rs:**
Implement `pub fn compute_score(findings: &[Finding]) -> String` that:

1. Sum severity weights: Critical=10, High=5, Medium=2, Low=1 (use the `score_weight()` method on Severity).
2. Map total to letter grade:
   - 0 points: "A+"
   - 1-5 points: "A"
   - 6-10 points: "B"
   - 11-20 points: "C"
   - 21-40 points: "D"
   - 41+ points: "F"
3. Return the grade as a String.

Implement `pub fn deduplicate_findings(findings: Vec<Finding>) -> Vec<Finding>` that:
1. Group findings by title (exact match).
2. For duplicates, keep the one with highest severity.
3. Append scanner names from duplicates into the kept finding's `raw_evidence` field (noting "Also found by: scanner_x").
4. Return deduplicated Vec.
