**Step 3: Create `src/metrics/middleware.rs` — HTTP metrics tracking middleware.**

Create the middleware function:
```rust
pub async fn track_http_metrics(req: Request, next: Next) -> impl IntoResponse
```

Implementation:
1. Extract route pattern from `req.extensions().get::<MatchedPath>()` — fall back to `req.uri().path().to_string()` if not available. Use `MatchedPath::as_str().to_owned()` (not `.to_string()` on the MatchedPath itself).
2. **Exclude paths** — if path equals `"/metrics"` or starts with `"/health"`, call `next.run(req).await` and return immediately (no metrics recorded). This is per user decision.
3. Clone `req.method()` and record `Instant::now()` before calling `next.run(req).await`.
4. After response, compute:
   - `latency = start.elapsed().as_secs_f64()`
   - `status_group = format!("{}xx", response.status().as_u16() / 100)` — per user decision: status groups not individual codes
5. Record metrics using the `metrics` crate macro syntax:
   - `metrics::counter!("http_requests_total", "method" => method.as_str().to_owned(), "endpoint" => path.clone(), "status" => status_group.clone()).increment(1);`
   - `metrics::histogram!("http_request_duration_seconds", "method" => method.as_str().to_owned(), "endpoint" => path).record(latency);`
6. Return the response.

Use imports: `axum::extract::{Request, MatchedPath}`, `axum::middleware::Next`, `axum::response::IntoResponse`, `std::time::Instant`

**Step 4: Create `src/api/metrics.rs` — /metrics endpoint handler with localhost check.**

**Step 3: Add metrics middleware layer to the API router.**

Add the HTTP metrics tracking middleware as a layer. It must run AFTER route matching (so MatchedPath is available) but capture all API routes. Place it as the first `.layer()` call (which means it executes last in the middleware chain — after CORS and tracing, wrapping the inner handler):

```rust
let app = Router::new()
    // API routes
    .route("/api/v1/scans", post(scans::create_scan))
    // ... other routes ...
    .layer(axum::middleware::from_fn(metrics::middleware::track_http_metrics))
    .layer(cors)
    .layer(trace_layer)
    .layer(middleware::from_fn(inject_request_id))
    .with_state(state.clone())
    // Health + metrics — merged AFTER layers, bypass tracing + metrics middleware
    .merge(health_router)
    .merge(metrics_router);
```

**IMPORTANT layer ordering:** Layers are applied bottom-to-top in Axum. The `track_http_metrics` middleware layer should be placed ABOVE `cors` and `trace_layer` so it wraps the inner handler closest to it. This means it runs AFTER route matching and request ID injection. The exact position:
- `.layer(from_fn(track_http_metrics))` — first layer listed (closest to routes)
- `.layer(cors)` — second
- `.layer(trace_layer)` — third
- `.layer(from_fn(inject_request_id))` — fourth (outermost)

This means execution order is: inject_request_id → trace_layer → cors → track_http_metrics → handler → track_http_metrics(response) → cors → trace_layer → inject_request_id. The metrics middleware wraps closest to the handler so MatchedPath is available and it captures the true handler latency.
