<task type="auto">
  <name>Task 1: Add dependencies and refactor container.rs to native binary execution</name>
  <files>Cargo.toml, src/scanners/container.rs</files>
  <action>
1. Add to Cargo.toml `[dependencies]`:
   - `which = "7"` (binary path resolution)
   - `tempfile = "3"` (temp file for JSON output)

2. Refactor `src/scanners/container.rs`:

   **Replace `is_docker_available()` with binary resolution functions:**

   Create `resolve_nuclei_binary() -> Option<PathBuf>`:
   - Check `NUCLEI_BINARY_PATH` env var first. If set but path doesn't exist, log warning.
   - Fall back to `which::which("nuclei")` for PATH lookup.
   - Return `None` if not found (enables graceful skip).

   Create `resolve_testssl_binary() -> Option<PathBuf>`:
   - Check `TESTSSL_BINARY_PATH` env var first. If set but path doesn't exist, log warning.
   - Fall back to `which::which("testssl.sh")` for PATH lookup.
   - Return `None` if not found.

   **Refactor `run_nuclei(target: &str)`:**
   - Replace Docker availability check with `resolve_nuclei_binary()`. If `None`, log warning "Nuclei binary not found, skipping Nuclei scan" and return `Ok(Vec::new())`.
   - Create a `NamedTempFile` for JSON output.
   - Build args for the native Nuclei binary (NOT Docker): `-u target -jsonl -silent -severity medium,high,critical -tags exposure,misconfig,cve -o <temp_path>`
   - Execute via `Command::new(nuclei_binary_path)` with `.args()`, `.stdout(Stdio::null())`, `.stderr(Stdio::piped())`.
   - Use `tokio::time::timeout(Duration::from_secs(120), ...)` for timeout.
   - After process completes, read JSON from temp file with `std::fs::read_to_string()`.
   - Parse with existing `parse_nuclei_output()` (no changes needed to parsers).
   - NamedTempFile auto-cleans on drop.

   **Refactor `run_testssl(target: &str)`:**
   - Replace Docker availability check with `resolve_testssl_binary()`. If `None`, log warning and return `Ok(Vec::new())`.
   - Create a `NamedTempFile` for JSON output.
   - Build args for native testssl.sh: `--jsonfile-pretty <temp_path> --quiet target`
   - Execute via `Command::new(testssl_binary_path)`.
   - Read JSON from temp file after completion.
   - Parse with existing `parse_testssl_output()`.

   **Remove `run_docker_container()` function** (no longer needed -- vibecode.rs has its own copy).

   **Update `ScannerError` enum:**
   - Rename `DockerNotAvailable` to `BinaryNotFound` (or keep both for backward compat -- prefer rename since nothing external uses it).
   - Rename `ContainerTimeout` to `ScanTimeout`.
   - Rename `ContainerError` to `ExecutionError`.
   - Update Display impl accordingly.

   **Keep all existing parser functions unchanged** (`parse_nuclei_output`, `parse_nuclei_finding`, `parse_testssl_output`, `parse_testssl_finding`, `get_testssl_remediation`).

   **Update tests:**
   - Rename `test_docker_availability` to `test_binary_resolution` -- test that resolve functions return `None` gracefully when binaries aren't installed.
   - Keep `test_nuclei_finding_parse` and `test_testssl_severity_mapping` unchanged (they test parsers, not execution).
  </action>

Run `cargo check` -- should compile without errors. Run `cargo test --lib scanners::container` -- parser tests should pass, binary resolution test should handle missing binaries gracefully.
  </verify>

  </done>

   - Remove the Docker-specific volume mount args (`-v`, volume_arg).
   - Create a `NamedTempFile` for JSON output.

   - Read JSON from temp file after completion.
   - Parse with existing `parse_vibecode_output()`.

   **Update `select_templates()`:**

   - This makes templates work regardless of where Nuclei runs (Docker or native).

   **Remove from vibecode.rs:**
   - `is_docker_available()` function (duplicate of container.rs version, no longer needed).
   - `run_docker_container()` function (duplicate, no longer needed).
