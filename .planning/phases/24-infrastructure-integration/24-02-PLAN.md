  <action>
    Human deploys all Phase 24 configuration changes to production via Ansible playbook.

    **Deploy to production server:**

    From the infrastructure/ directory, run the resume-app playbook which deploys all templates (Nginx, Docker Compose, systemd, env) and installs the metrics agent:

    ```bash
    cd infrastructure
    ansible-playbook playbooks/resume-app.yml -v -e "droplet_ip=YOUR_DROPLET_IP" --ask-vault-pass
    ```

    The playbook will:
    1. Deploy updated Nginx config with /metrics restriction
    2. Deploy updated Docker Compose with STOPSIGNAL and stop_grace_period
    3. Deploy updated .env with LOG_FORMAT=json and SHUTDOWN_TIMEOUT=90
    4. Deploy updated systemd service with TimeoutStopSec=95s
    5. Install DigitalOcean metrics agent (if not already installed)
    6. Restart services to apply changes

    **After playbook completes, restart the service to pick up all changes:**

    ```bash
    # SSH into the server
    ssh -p 2222 deploy@YOUR_DROPLET_IP

    # Restart the service (applies new systemd config + recreates containers)
    sudo systemctl daemon-reload
    sudo systemctl restart trustedge

    # Wait for services to come up
    sleep 30
    ```

    Type "deployed" when the playbook and restart complete successfully.
  </action>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify all observability features in production</name>
  <action>
    Human verifies all v1.4 observability infrastructure is working: JSON logging, metrics endpoint security, graceful shutdown coordination, Docker log rotation, and DigitalOcean metrics agent.

    SSH into the production server and run these verification commands:

    **1. INF-01 — Metrics agent running:**
    ```bash
    systemctl is-active do-agent
    # Expected: active
    ```

    **2. INF-02 — /metrics restricted externally:**
    From your LOCAL machine (not the server):
    ```bash
    curl -s -o /dev/null -w "%{http_code}" https://YOUR_DOMAIN/metrics
    # Expected: 403
    ```
    From the SERVER:
    ```bash
    curl -s http://127.0.0.1:3000/metrics | head -5
    # Expected: Prometheus metrics text starting with "# HELP" or "# TYPE"
    ```

    **3. INF-03 — Docker Compose graceful shutdown:**
    ```bash
    docker inspect $(docker ps -q --filter name=backend) --format='{{.HostConfig.LogConfig}}'
    # Expected: {json-file map[max-file:3 max-size:10m]}

    docker inspect $(docker ps -q --filter name=backend) --format='{{.Config.StopSignal}}'
    # Expected: SIGTERM

    grep stop_grace_period /opt/trustedge/docker-compose.yml
    # Expected: stop_grace_period: 90s (backend) and stop_grace_period: 10s (frontend)
    ```

    **4. INF-04 — systemd timeout:**
    ```bash
    systemctl show trustedge.service -p TimeoutStopUSec
    # Expected: TimeoutStopUSec=1min 35s (which is 95s)
    ```

    **5. INF-05 — JSON logging:**
    ```bash
    grep LOG_FORMAT /opt/trustedge/.env
    # Expected: LOG_FORMAT=json

    docker logs $(docker ps -q --filter name=backend) --tail 5
    # Expected: JSON formatted log lines ({"timestamp":..., "level":..., ...})
    ```

    **6. DigitalOcean Control Panel:**
    Visit DigitalOcean dashboard > Droplets > your droplet > Graphs tab.
    Verify CPU, Memory, and Disk graphs are populated with data.

    **7. Overall health:**
    ```bash
    curl -s http://127.0.0.1:3000/health
    # Expected: "ok" with 200 status

    curl -s http://127.0.0.1:3000/health/ready | python3 -m json.tool
    # Expected: JSON with db_connected: true, status: "ready"
    ```
  </action>
  <verify>All 7 verification checks pass on the production server.</verify>
  <done>All observability features verified working in production. v1.4 milestone complete.</done>
  <resume-signal>Type "approved" if all checks pass, or describe any failures</resume-signal>
</task>
