**2. Create `src/api/health.rs`** with:

**Response types** (all derive Serialize, Clone):
- `LivenessResponse { status: String }` — always returns `{ "status": "ok" }`
- `ScanCapacity { active: usize, max: usize }` — in-flight vs limit
- `ReadinessResponse { db_connected: bool, scan_capacity: ScanCapacity, status: String }` — exactly three fields per user decision. `status` is one of "healthy", "degraded", "unhealthy".

**HealthCache struct** (Clone):
- `inner: Arc<std::sync::Mutex<Option<(std::time::Instant, ReadinessResponse)>>>`
- Methods:
  - `new() -> Self` — creates with `None` initial state
  - `get_cached(&self, ttl: Duration) -> Option<ReadinessResponse>` — returns clone if within TTL, None otherwise. Clone inside lock, drop guard before returning.
  - `update(&self, response: ReadinessResponse)` — stores with current `Instant::now()`

**Liveness handler:**
```rust
pub async fn health_liveness() -> Json<LivenessResponse> {
    Json(LivenessResponse { status: "ok".to_string() })
}
```
No I/O, no dependencies, returns immediately. Per user decision: JSON `{ "status": "ok" }` for consistency.

**Readiness handler:**
```rust
pub async fn health_readiness(
    State(state): State<AppState>,
) -> (StatusCode, Json<ReadinessResponse>)
```

The handler needs access to AppState (for pool and orchestrator) AND HealthCache. Since both routes are placed after `.with_state(state)`, the HealthCache needs to be part of AppState or passed via a merged state. The simplest approach: **Add `health_cache: HealthCache` field to AppState** in `src/api/scans.rs`. This avoids complex nested state extraction.

Readiness logic:
1. Check cache — call `health_cache.get_cached(Duration::from_secs(5))`. If Some, compute status code from cached response and return.
2. Get scan capacity — call `state.orchestrator.get_capacity()` to get `(active, max)`.
3. Check DB connectivity with latency — measure `Instant::now()` before/after `sqlx::query("SELECT 1").fetch_one(&state.pool).await`. Read `HEALTH_DB_LATENCY_THRESHOLD_MS` env var (default 50ms, parse with `.ok().and_then(|s| s.parse::<u64>().ok()).unwrap_or(50)`).
4. Determine status:
   - DB unreachable (Err) → `db_connected: false`, `status: "unhealthy"`
   - DB latency > threshold → `db_connected: true`, `status: "degraded"`
   - DB latency <= threshold → `db_connected: true`, `status: "healthy"`
5. Build `ReadinessResponse` with all three fields.
6. Update cache with the new response.
7. Map status to HTTP status code: "healthy" → 200, "degraded" → 429, "unhealthy" → 503.
8. Return `(status_code, Json(response))`.

IMPORTANT: Do NOT hold `std::sync::Mutex` guard across `.await` points. The cache check and update are synchronous operations — lock, clone/write, drop guard, then proceed to async work.

KEY DETAIL: The health routes are placed AFTER `.with_state(state)` to bypass tracing layers but BEFORE `.into_make_service_with_connect_info()`. Since health routes are added after `.with_state()`, they won't have access to AppState by default.

**Fix:** The readiness handler needs AppState. Two approaches:
- (A) Add health routes BEFORE `.layer()` calls but use a route-specific bypass — complex.
- (B) Create a separate nested router for health with its own state, merged into the main router.

The cleanest approach: Keep health routes in the main router BEFORE the layers, but use Axum's `nest` or route ordering to exclude them from tracing. Actually, looking at the existing pattern — the `/health` route is already AFTER `.with_state(state)`. In Axum, routes added after `.with_state()` don't receive the state. So the current inline `|| async { "ok" }` works because it doesn't need state.

For `/health/ready` which NEEDS state, use this pattern: Create a separate health router with state, and merge it after the layers:

```rust
// Health check router — separate from traced routes
let health_router = Router::new()
    .route("/health", get(health::health_liveness))
    .route("/health/ready", get(health::health_readiness))
    .with_state(state.clone());

let app = Router::new()
    // API routes
    .route("/api/v1/scans", post(scans::create_scan))
    // ... other routes ...
    .layer(cors)
    .layer(trace_layer)
    .layer(middleware::from_fn(inject_request_id))
    .with_state(state)
    // Health checks — merged AFTER layers, bypass tracing
    .merge(health_router)
    .into_make_service_with_connect_info::<SocketAddr>();
```

This ensures:
1. Health routes bypass tracing (merged after layers)
2. Health routes have access to AppState (via their own `.with_state()`)
3. Liveness remains stateless (handler ignores state)
4. Readiness can access pool and orchestrator through AppState
