29. All other severity/grade dark mode pairs — must be >= 4.5:1

**Step 2: Calculate contrast ratios.**

For each pair, calculate the contrast ratio using the WCAG 2.1 relative luminance formula. OKLch L (lightness) values can be used to approximate contrast: ratio = (L_lighter + 0.05) / (L_darker + 0.05).

For more accurate results, convert OKLch to sRGB then calculate relative luminance per WCAG spec.

Use a script or manual calculation. For OKLch approximate contrast:
- L=0.98 on L=0.13 → (0.98+0.05)/(0.13+0.05) = 5.7:1 (passes)
- L=0.45 on L=0.98 → (0.98+0.05)/(0.45+0.05) = 2.1:1 (FAILS for normal text)

**Step 3: Fix any violations.**

If a pair fails 4.5:1:
- For text on light backgrounds: darken the text (lower L value)
- For text on dark backgrounds: lighten the text (raise L value)
- Adjust the primitive OKLch L value in globals.css :root or the dark override block
- Re-verify the pair and all other pairs affected by the change

**Common trouble spots (likely to fail, check carefully):**
- text-tertiary (gray-500, L≈0.55) on surface-primary (white, L=1.0): approximately 4.6:1 — borderline, may need adjustment
- text-muted (gray-400, L≈0.71) on surface-primary: approximately 3.0:1 — only acceptable for decorative/large text
- text-secondary (gray-400 in dark mode) on surface-secondary (gray-900): may be tight
- severity-medium-text (yellow-600) on severity-medium-bg (yellow-100): yellow is notoriously hard for contrast

**Step 4: Update globals.css with corrected values.**

Only modify primitive OKLch L values where needed. Keep hue and chroma consistent to maintain visual identity.
