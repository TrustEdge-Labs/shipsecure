**infrastructure/templates/trustedge.nginx.conf.j2** — Nginx site configuration template:

Create a Jinja2 template with two server blocks:

**Server block 1 — HTTP to HTTPS redirect:**
```nginx
server {
    listen 80;
    listen [::]:80;
    server_name {{ domain_name }};

    # Certbot ACME challenge directory (needed for initial cert)
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$server_name$request_uri;
    }
}
```

**Server block 2 — HTTPS with reverse proxy:**
```nginx
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name {{ domain_name }};

    # SSL certificates — managed by Certbot
    ssl_certificate /etc/letsencrypt/live/{{ domain_name }}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/{{ domain_name }}/privkey.pem;

    # SSL configuration — Mozilla Intermediate profile
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:10m;
    ssl_session_tickets off;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    resolver 1.1.1.1 1.0.0.1 valid=300s;
    resolver_timeout 5s;

    # Security headers
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # Backend API — proxy to Rust/Axum backend
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection "";

        # Scan timeouts — scans take 30s-3min
        proxy_connect_timeout 10s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;

        # Request body limits for API
        client_max_body_size 1m;
    }

    # Frontend — proxy to Next.js
    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection "";

        # Standard timeouts for frontend
        proxy_connect_timeout 10s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

**infrastructure/tasks/nginx.yml** — Nginx installation and configuration:

1. **Install Nginx** — `apt: name=nginx state=present`
2. **Create ACME challenge directory** — `file: path=/var/www/certbot state=directory`
3. **Deploy initial Nginx config (HTTP only)** — Write a temporary HTTP-only server block to `/etc/nginx/sites-available/trustedge` that handles ACME challenges and redirects to HTTPS. This is needed because the full config references SSL certs that don't exist yet.

The initial config should be:
```nginx
server {
    listen 80;
    listen [::]:80;
    server_name {{ domain_name }};

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 200 'TrustEdge Audit — waiting for SSL certificate';
        add_header Content-Type text/plain;
    }
}
```

4. **Enable site via symlink** — `file: src=/etc/nginx/sites-available/trustedge dest=/etc/nginx/sites-enabled/trustedge state=link`
5. **Remove default Nginx site** — `file: path=/etc/nginx/sites-enabled/default state=absent`
6. **Test Nginx config** — `command: nginx -t` with `changed_when: false`
7. **Enable and start Nginx** — `systemd: name=nginx enabled=yes state=restarted`

IMPORTANT: Deploy the HTTP-only config FIRST. Certbot task will obtain the cert, then we deploy the full HTTPS config (trustedge.nginx.conf.j2 template) AFTER certbot succeeds. This avoids the chicken-and-egg problem where the full config references SSL certs that don't exist yet.

**infrastructure/tasks/certbot.yml** — SSL certificate provisioning:

1. **Install Certbot via snap** — Use `snap` module:
   - `snap: name=certbot classic=yes state=present`

2. **Create Certbot symlink** — `file: src=/snap/bin/certbot dest=/usr/local/bin/certbot state=link`

3. **Check if certificate already exists** — `stat: path=/etc/letsencrypt/live/{{ domain_name }}/fullchain.pem` — Register as `cert_exists`

4. **Obtain SSL certificate** — `command: certbot --nginx --non-interactive --agree-tos --email {{ admin_email }} --domain {{ domain_name }} --redirect`
   - `when: not cert_exists.stat.exists` (idempotent — skip if cert already obtained)
   - Use `creates: /etc/letsencrypt/live/{{ domain_name }}/fullchain.pem` as additional safeguard

5. **Deploy full HTTPS Nginx config** — Use `template` module:
   - `src: ../templates/trustedge.nginx.conf.j2`
   - `dest: /etc/nginx/sites-available/trustedge`
   - `notify: Reload Nginx`
   - This replaces the temporary HTTP-only config with the full reverse proxy + SSL config

6. **Test Nginx config** — `command: nginx -t` with `changed_when: false`

7. **Reload Nginx** — `systemd: name=nginx state=reloaded`

8. **Test certificate renewal** — `command: certbot renew --dry-run` with `changed_when: false`

9. **Define handler** — `Reload Nginx: systemd: name=nginx state=reloaded`

NOTE: Certbot's snap installation includes a systemd timer for automatic renewal (runs twice daily). No need to create a separate cron job. The Nginx plugin handles cert placement and auto-reloads Nginx on renewal.

**infrastructure/templates/trustedge.service.j2** — Systemd service unit template:

```ini
[Unit]
Description=TrustEdge Audit Application
Requires=docker.service
After=docker.service network-online.target
Wants=network-online.target

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory={{ app_directory }}
EnvironmentFile={{ app_directory }}/.env
ExecStart=/usr/bin/docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
ExecStop=/usr/bin/docker compose -f docker-compose.yml -f docker-compose.prod.yml down
ExecReload=/usr/bin/docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
TimeoutStartSec=600
TimeoutStopSec=30
Restart=on-failure
RestartSec=15
User={{ deploy_user }}
Group={{ deploy_user }}

[Install]
WantedBy=multi-user.target
```

Key design decisions:
- `Type=oneshot` + `RemainAfterExit=yes`: docker compose runs once and exits, systemd tracks it as active
- `EnvironmentFile`: loads production .env for docker compose variable substitution
- `--build` flag in ExecStart: rebuilds images on deploy (git pull + systemctl restart trustedge)
- `TimeoutStartSec=600`: 10 minutes for first build (Docker image build can be slow)
- `Restart=on-failure`: auto-restart if compose fails (but NOT on clean stop)
- `User/Group=deploy`: runs as non-root deploy user
- `ExecReload`: allows `systemctl reload trustedge` for zero-confusion updates

**infrastructure/tasks/app.yml** — Application deployment:

1. **Create application directory** — `file: path={{ app_directory }} state=directory owner={{ deploy_user }} group={{ deploy_user }} mode='0755'`

2. **Clone application repository** — Use `git` module:
   - `repo: "{{ app_git_repo }}"`
   - `dest: "{{ app_directory }}"`
   - `version: main`
   - `force: yes`
   - `become_user: "{{ deploy_user }}"`

3. **Deploy production .env file** — Use `template` module:
   - `src: ../templates/env.production.j2`
   - `dest: "{{ app_directory }}/.env"`
   - `owner: "{{ deploy_user }}"`
   - `group: "{{ deploy_user }}"`
   - `mode: '0600'` (secrets file, readable only by owner)

4. **Install Nuclei binary** — Multi-step:
   a. Check if Nuclei already installed: `stat: path=/usr/local/bin/nuclei` → register `nuclei_installed`
   b. Get latest Nuclei version from GitHub API: `uri: url=https://api.github.com/repos/projectdiscovery/nuclei/releases/latest return_content=yes` → register `nuclei_release`, `when: not nuclei_installed.stat.exists`
   c. Set Nuclei version fact: `set_fact: nuclei_version={{ nuclei_release.json.tag_name | regex_replace('^v', '') }}`
   d. Download Nuclei: `get_url: url=https://github.com/projectdiscovery/nuclei/releases/download/v{{ nuclei_version }}/nuclei_{{ nuclei_version }}_linux_amd64.zip dest=/tmp/nuclei.zip`
   e. Install unzip if needed: `apt: name=unzip state=present`
   f. Extract to /usr/local/bin: `unarchive: src=/tmp/nuclei.zip dest=/usr/local/bin/ remote_src=yes creates=/usr/local/bin/nuclei`
   g. Set executable permissions: `file: path=/usr/local/bin/nuclei mode='0755'`
   h. Clean up: `file: path=/tmp/nuclei.zip state=absent`

5. **Verify Nuclei installation** — `command: nuclei -version` with `changed_when: false`

6. **Pre-pull Docker images** — `command: docker compose -f docker-compose.yml -f docker-compose.prod.yml pull` in `{{ app_directory }}` as `{{ deploy_user }}`. Use `changed_when: false` since this is a preparation step.
