  <action>
Create `frontend/components/page-container.tsx` as a simple server-compatible layout wrapper. The component accepts `children`, an optional `maxWidth` prop (defaults to `max-w-4xl`), and an optional `className` for additional overrides.

```tsx
import { type ReactNode } from 'react'
import { cn } from '@/lib/utils'

interface PageContainerProps {
  children: ReactNode
  maxWidth?: string
  className?: string
}

export function PageContainer({
  children,
  maxWidth = 'max-w-4xl',
  className,
}: PageContainerProps) {
  return (
    <div className={cn('container mx-auto px-4', maxWidth, className)}>
      {children}
    </div>
  )
}
```

Check if `@/lib/utils` with a `cn` helper exists in the project. If it does, import from there. If no `cn` utility exists, replace with a simple string join:

```tsx
const classes = ['container mx-auto px-4', maxWidth, className]
  .filter(Boolean)
  .join(' ')
return <div className={classes}>{children}</div>
```

This component is NOT a client component — no `'use client'` directive. It is a pure layout wrapper.
