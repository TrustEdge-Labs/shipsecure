```rust
async fn reject_scans_during_shutdown(
    State(state): State<AppState>,
    request: axum::extract::Request,
    next: axum::middleware::Next,
) -> axum::response::Response {
    // Only reject scan creation endpoints during shutdown

        return (

            axum::Json(error_body),
        ).into_response();

    next.run(request).await
}
```

**1. Add shutdown_signal() function:**

```rust
async fn shutdown_signal() {
    let ctrl_c = async {
        tokio::signal::ctrl_c()
            .await
            .expect("failed to install Ctrl+C handler");
    };

    #[cfg(unix)]
    let terminate = async {
        tokio::signal::unix::signal(tokio::signal::unix::SignalKind::terminate())
            .expect("failed to install signal handler")
            .recv()
            .await;
    };

    #[cfg(not(unix))]
    let terminate = std::future::pending::<()>();

    tokio::select! {
        _ = ctrl_c => {
            tracing::info!("Received SIGINT, initiating graceful shutdown");
        },
        _ = terminate => {
            tracing::info!("Received SIGTERM, initiating graceful shutdown");
        },
    }
}
```

Per user decision: SIGTERM and SIGINT handled identically.

**4. Add shutdown middleware as outermost layer on API routes:**

The middleware must run FIRST (outermost = added LAST in Axum's layer chain). Add it after the existing layers but before `.with_state()`:

```rust
let app = Router::new()

    .route("/api/v1/scans", post(scans::create_scan))
    // ... other routes ...
    .layer(axum::middleware::from_fn_with_state(state.clone(), reject_scans_during_shutdown))
    .layer(axum::middleware::from_fn(metrics::middleware::track_http_metrics))
    .layer(cors)
    .layer(trace_layer)
    .layer(middleware::from_fn(inject_request_id))
    .with_state(state)
    .merge(health_router)
    .merge(metrics_router)
    .into_make_service_with_connect_info::<SocketAddr>();
```

IMPORTANT: Axum layers execute in reverse order of addition. The shutdown middleware is added FIRST in the chain (closest to routes), which means it executes LAST in middleware ordering. Actually wait -- we want it to execute FIRST (before other middleware). In Axum, layers wrap from inside out: the first `.layer()` is the innermost layer. So the LAST `.layer()` executes FIRST. Therefore, `reject_scans_during_shutdown` should be the LAST layer added (just before `.with_state()`), which means it wraps everything and executes first.

Revised ordering:
```rust
let app = Router::new()
    .route("/api/v1/scans", post(scans::create_scan))
    // ... other routes ...
    .layer(axum::middleware::from_fn(metrics::middleware::track_http_metrics))
    .layer(cors)
    .layer(trace_layer)
    .layer(middleware::from_fn(inject_request_id))
    .layer(axum::middleware::from_fn_with_state(state.clone(), reject_scans_during_shutdown))
    .with_state(state)
    .merge(health_router)
    .merge(metrics_router)
```

This way `reject_scans_during_shutdown` is outermost and runs first.

**5. Replace `axum::serve(listener, app).await` with graceful shutdown coordination:**

```rust
// Log startup with shutdown config
tracing::info!(
    addr = %addr,
    shutdown_timeout_seconds = shutdown_timeout.as_secs(),
    "Listening"
);

// Serve with graceful shutdown
axum::serve(listener, app)
    .with_graceful_shutdown(shutdown_signal())
    .await
    .expect("Server error");

// -- At this point, shutdown signal was received and HTTP server stopped accepting --

// Initiate orchestrator shutdown (close tracker + cancel token)
tracing::info!(
    timeout_seconds = shutdown_timeout.as_secs(),
    "Graceful shutdown initiated, draining in-flight scans"
);
orchestrator.initiate_shutdown();

// Periodic logging + wait with timeout
let start = std::time::Instant::now();
let log_interval = Duration::from_secs(5);

loop {
    let remaining = shutdown_timeout.checked_sub(start.elapsed()).unwrap_or(Duration::ZERO);
    if remaining.is_zero() {
        // Timeout expired
        let (active, max) = orchestrator.get_capacity();
        let queued = 0; // After close(), no new tasks queued
        tracing::warn!(
            active_scans = active,
            elapsed_seconds = start.elapsed().as_secs(),
            timeout_seconds = shutdown_timeout.as_secs(),
            "Shutdown forced: {} scans remaining after {}s",
            active,
            start.elapsed().as_secs()
        );
        break;
    }

    // Wait for either: all tasks done, or next log interval
    let wait_duration = remaining.min(log_interval);
    tokio::select! {
        _ = orchestrator.wait_for_drain() => {
            // All tasks completed gracefully -- no summary log per user decision
            break;
        }
        _ = tokio::time::sleep(wait_duration) => {
            // Log progress
            let (active, _max) = orchestrator.get_capacity();
            if active == 0 {
                break; // Clean exit
            }
            tracing::info!(
                active_scans = active,
                elapsed_seconds = start.elapsed().as_secs(),
                timeout_seconds = shutdown_timeout.as_secs(),
                "shutdown_progress"
            );
        }
    }
}

// Exit with code 0 -- per user decision, clean exit prevents systemd restart
std::process::exit(0);
```

Per user decisions:
- Periodic progress every 5s (within 5-10s range, Claude's discretion)
- Structured fields: active_scans, elapsed_seconds, timeout_seconds
- INFO for progress, WARN for forced timeout
- Final summary ONLY on forced shutdown
- Normal clean shutdowns: no summary (periodic logs sufficient)
- exit(0) after forced shutdown

NOTE on wait_for_drain: The `wait_for_drain()` call resolves when ALL tracked tasks complete. But we need to call it fresh each loop iteration. Actually, `TaskTracker::wait()` returns a future that completes when there are no more tracked tasks. We can hold one future across loop iterations. Restructure to use a single `wait()` future:

```rust
let drain_future = orchestrator.wait_for_drain();
tokio::pin!(drain_future);

loop {
    // ... (timeout check)
    tokio::select! {
        _ = &mut drain_future => { break; }
        _ = tokio::time::sleep(wait_duration) => { /* log progress */ }
    }
}
```
