  <files>frontend/components/scan-history-table.tsx</files>
  <action>
In `frontend/components/scan-history-table.tsx`, the clickable row branch (when `isClickable === true`) renders:
1. An overlay `<Link>` inside the first `<td>` with `aria-label` covering the full row
2. An explicit "View" `<Link>` in the last `<td>`

Screen readers encounter two links to the same `/results/${scan.results_token}` URL. Fix: remove the explicit "View" `<Link>` from the action `<td>` in the clickable-row branch (the overlay link already provides full-row navigation with a descriptive aria-label). The `<td>` cell should remain in the markup for table column alignment — just leave it empty (or show nothing) in the clickable case.

**Current action `<td>` in the clickable branch (approx lines 192-199):**
```tsx
<td className="py-3 relative z-10">
  <Link
    href={`/results/${scan.results_token}`}
    className="text-xs font-medium text-brand-primary hover:underline"
  >
    View
  </Link>
</td>
```

**Replace with:**
```tsx
<td className="py-3" aria-hidden="true" />
```

This keeps column count consistent (table layout remains valid) while eliminating the duplicate link from the accessibility tree. `aria-hidden="true"` on the empty cell prevents screen readers from announcing an empty cell.

Do NOT change:
- The overlay Link in the first `<td>` — it is the correct primary navigation mechanism with its aria-label
- The non-clickable row branch (isExpired / isFailed rows still show the "Expired" badge in the action cell as before)
- The mobile card layout — it already uses a single `<Link>` wrapping the whole card
- Any other part of the component
