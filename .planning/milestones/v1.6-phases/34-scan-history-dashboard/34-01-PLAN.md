Use `String` for `status` (not `ScanStatus` enum) — the query casts `s.status::text`. Use `i64` for counts (PostgreSQL COUNT returns bigint). Use `NaiveDateTime` for timestamps with `::timestamp` cast to match existing patterns in this file.

2. **`get_user_scan_history`** — paginated query for completed/failed scans:

```rust
#[allow(dead_code)]
pub async fn get_user_scan_history(
    pool: &PgPool,
    clerk_user_id: &str,
    limit: i64,
    offset: i64,
) -> Result<Vec<ScanHistoryRow>, sqlx::Error>
```

SQL: `LEFT JOIN findings f ON f.scan_id = s.id` with `COUNT(CASE WHEN f.severity = 'critical' THEN 1 END)` for each severity level. Filter `WHERE s.clerk_user_id = $1 AND s.status NOT IN ('pending', 'in_progress')`. GROUP BY all non-aggregated columns. ORDER BY `CASE WHEN s.expires_at IS NULL THEN 1 ELSE 0 END ASC, s.expires_at ASC NULLS LAST, s.created_at DESC` (expiring soonest first per locked decision). Cast `s.expires_at::timestamp` and `s.created_at::timestamp` and `s.status::text` for Rust type mapping.

3. **`count_user_scans_history`** — total count for pagination:

```rust
#[allow(dead_code)]
pub async fn count_user_scans_history(
    pool: &PgPool,
    clerk_user_id: &str,
) -> Result<i64, sqlx::Error>
```

SQL: `SELECT COUNT(*) FROM scans WHERE clerk_user_id = $1 AND status NOT IN ('pending', 'in_progress')`. Returns `(i64,)` tuple pattern consistent with existing count functions in this file.

4. **`get_user_active_scans`** — non-paginated list of in-progress/pending scans:

```rust

    pool: &PgPool,
```

1. **Create `src/api/users.rs`** (new file):

```rust
use axum::extract::{Query, State};
use axum::Json;
use axum_jwt_auth::Claims;
use serde::Deserialize;
use serde_json::json;

use crate::api::auth::ClerkClaims;
use crate::api::errors::ApiError;
use crate::api::scans::AppState;
use crate::db;

#[derive(Deserialize)]
pub struct ScanHistoryQuery {
    pub page: Option<u32>,
}

pub async fn get_user_scans(
    State(state): State<AppState>,
    Claims { claims, .. }: Claims<ClerkClaims>,
    Query(params): Query<ScanHistoryQuery>,
) -> Result<Json<serde_json::Value>, ApiError> {
    let page = params.page.unwrap_or(1).max(1) as i64;
    let per_page: i64 = 10;
    let offset = (page - 1) * per_page;

    let (scans, active_scans, total) = tokio::try_join!(
        db::scans::get_user_scan_history(&state.pool, &claims.sub, per_page, offset),
        db::scans::get_user_active_scans(&state.pool, &claims.sub),
        db::scans::count_user_scans_history(&state.pool, &claims.sub),
    )?;

    let total_pages = if total == 0 { 0 } else { (total + per_page - 1) / per_page };

    Ok(Json(json!({
        "scans": scans,
        "active_scans": active_scans,
        "total": total,
        "page": page,
        "per_page": per_page,
        "total_pages": total_pages,
    })))
}
```

Uses `Claims<ClerkClaims>` extractor (mandatory auth — anonymous callers get 401). Uses `tokio::try_join!` to run all three DB queries concurrently. The `?` propagates `sqlx::Error` which converts to `ApiError::InternalError` via the existing `From<sqlx::Error>` impl in `errors.rs`. Per-page is hardcoded at 10 per locked decision.
