<objective>
Implement server-side results gating so that anonymous API callers see high/critical findings with stripped description and remediation fields, while authenticated scan owners see full details.

Purpose: Prevent curl bypass of content gating (OWASP A01). The server must strip sensitive fields before the response leaves — never rely on the frontend to hide data.
Output: Modified results API that accepts optional JWT, computes owner_verified, strips high/critical finding details for non-owners, and returns gated/owner_verified fields.
</objective>

1. Add an `extract_optional_clerk_user` async function to `src/api/results.rs`:

```rust
use axum::http::header::AUTHORIZATION;

async fn extract_optional_clerk_user(
    state: &AppState,
    headers: &axum::http::HeaderMap,
) -> Option<String> {
    let auth_value = headers.get(AUTHORIZATION)?.to_str().ok()?;
    let token = auth_value.strip_prefix("Bearer ")?.trim();
    if token.is_empty() { return None; }
    let token_data = state.jwt_decoder.decode(token).await.ok()?;
    Some(token_data.claims.sub)
}
```

This manually extracts the Bearer token and calls `state.jwt_decoder.decode()` directly. Do NOT use `Claims<ClerkClaims>` as a handler parameter — it would reject all anonymous requests with 401.
